Linux系统管理

进程管理

  • 简介
    • 进程简介
      • 正在执行的程序或命令
      • 每个进程都是运行实体
      • 有自己地址空间
      • 占用一定系统资源
    • 进程管理的作用
      • 判断服务器健康状态：top
      • 查看系统中所有进程：ps，pstree
      • 杀死进程（正常关闭失效时才考虑）：kill
  • 进程查看
    • PS命令：查看所有进程
      • ps aux
        • 查看系统当前所有进程，BSD操作系统格式（即Unix格式
        • a：所有前台进程
        • x：所有后台进程
        • u：进程由哪个用户触发
      • ps -le
        • 查看所有进程，Linux标准格式
        • l：显示详细信息
        • e：所有进程的意思
      • 输出结果分析
        • 一条记录为一个进程
        • USER：当前进程由哪个用户产生
        • PID：进程ID
          • init进程pid为1
          • 所有进程的父进程
        • %CPU：占用CPU的百分比
        • %MEM：占用的物理内存百分比
        • VSZ：占用虚拟内存的大小，单位KB
        • RSS：占用实际物理内存的大小，单位KB
        • TTY：是在哪个终端运行的
          • 很多系统进程的tty为？：表示内核调用
          • tty1-tty7：本地控制台终端
          • tty1-tty6: 本地字符界面终端
          • tty7:本地图形终端
          • pts/0-256：虚拟终端
          • ctrl+ alt+ F1/2：切换终端
        • STAT：进程状态
          • R：运行
          • S：睡眠
          • T：停止状态
          • s：包含子进程
          • +：位于后台
        • START：进程启动时间
        • TIME：占用CPU的运行时间，而非系统时间
        • COMMAND：产生此进程的命令
    • top命令：判断服务器健康状态
      • top 或top -d
        • 默认三秒刷新
        • d参数指定top每隔几秒更新
      • 交互模式下可用命名
        • ？或h：帮助信息
        • P：以CPU使用率排序，默认
        • M：以内存使用率排序
        • N：以pid排序
        • q：退出top
      • 输出结果分析
        • 第一行：任务队列信息：top
          • 系统当前时间
          • 系统已经运行的时间
          • 当前已登录的用户数
          • 系统在之前1分钟、5分钟、15分钟的平均负载
            • 小于1，负载较小
            • 大于1，超出负荷
            • 判断标准也取决于CPU核数，单核的以1来判断
        • 第二行：进程信息：Tasks
          • 系统中进程总数
          • 正在运行的进程数
          • 睡眠的进程数
          • 正在停止的进程
          • 僵尸进程
            • 表示正在终止，但还未终止完全
            • zombie关键字
            • 如果不是0，需手工检查僵尸进程
        • 第三行：CPU信息：%Cpus
          • 0.1%us：用户模式占用的CPU百分比
          • 0.1%sy：系统模式占用的CPU百分比
          • 0.0%ni：改变过优先级的用户进程占用的CPU百分比
          • 99.7%id：空闲CPU的占用百分比
            • 剩余CPU资源
            • 最主要看zhegeq
          • 0.1%wa：等待输入输出的进程的…
          • %hi：硬中断请求服务占用的…
          • %si：软中断请求服务占用的…
          • %st：虚拟时间百分比。当有虚拟机时，虚拟CPU等待实际CPU的时间百分比
        • 第四行：物理内存信息：MeM
          • total：物理内存总量，单位KB
          • used：已使用的
          • free：空闲的
          • buffers：作为缓冲的
        • 第五行：交换分区信息：Swap
          • total，used，free
          • cached：作为缓存的交换分区的大小
        • 重点关注
          • 第一行的平均负载
          • 第三行的CPU剩余
          • 第四行的空闲物理内存
    • pstree：查看进程树
      • -p：显示进程PID
      • -u：显示进程所属用户
  • 终止管理
    • kill命令
      • kill [信号] PID
        • 按照进程ID杀死进程
        • 常见重启或强制结束，即1或9
      • kill -l：查看可用的进程信号
        • 1:SIGHUP：重启进程：立即关闭进程，重新读取配置文件后重启
        • 9:SIGKILL：强制终止进程
        • 15:SIGTERM：正常结束进程，kill的默认信号
      • 杀死单个进程
    • killall命令
      • killall [选项] [信号] 进程名
        • 按照进程名杀死进程
        • -i：交互式，询问是否杀死某进程
        • -I：忽略进程名大小写
      • 可按照父进程名杀死多个子进程
    • pkill命令
      • pkill [选项] [信号] 进程名
        • 按照进程名杀死进程
        • -t：按照终端号踢出用户
          • w：查询本机已经登录的用户
          • pkill -t -9 pts/1 ：强制杀死从pts/1虚拟终端登录的进程
      • 按照终端号杀死对个进程

工作管理

  • 进程可放入后台（类似win最小化
    • 1，命令后加&
      • 放入后台依然运行
      • 如tar -zcf etc.tar.gz /etc &
    • 2，命令执行过程中，按ctrl+z（+c会强制终止）
      • 放入后台暂停运行
      • 如top命令时
  • 查看后台工作：jobs或jobs -l
    • l：显示工作的pid
    • 每条记录的第一个字段为工作号
    • +：最近一个放入后台的工作，默认优先恢复
    • -：倒数第二个放入后台的工作，默认第二个恢复
    • 第三个没有+、-等符号
  • 后台暂停的工作恢复到前台执行：fg %工作号
    • %工作号：指定要恢复的工作
    • %可省略
    • 工作号和pid是有区别的
  • 后台暂停的工作恢复到后台执行：bg %工作号
    • 若此命令是top（等与用户交互的命令），是不能在后台运行的
    • 与用户交互的命名如top，不能在后台运行，会失去其交互的意义，必须在前台运行来与用户交互
    • 如top、vi

系统资源查看

  • vmstat：监控系统资源
    • vmstat+刷新延时+刷新次数
    • 如，vmstat 1 3（监控3次，间隔1秒
    • 类似top，但比较简洁
  • dmesg：显示开机时内核检测信息
    • 如，dmesg | grep CPU
    • 开机时会显示，此命令可再次显示其信息
  • free：查看内存使用状态
    • free -b/k/m/g
    • b：以字节为单位显示
    • k：以KB为单位，默认也如此
    • m：以MB
    • g：以GB
  • 查看CPU信息
    • cat /proc/cpuinfo
    • 或者利用dmesg
  • uptime命令
    • 显示系统启用时间和平均负载
    • top命令的第一行结果
    • w命令也可看到此数据
  • uname命令：查看系统和内核信息
    • -a：查看系统所有相关信息
    • -r：查看内核版本
    • -s：查看内核名称
  • 判断系统当前位数
    • file /bin/ls
    • 来间接查看
  • 查询当前系统发行版本
    • lsb_release -a
  • lsof：列出某进程调用或打开的文件的信息
    • -c 字符串：只列出以指定字符串开头的进程打开的…
    • -u 用户名：只列出某个用户的进程打开的…
    • -p pid：只列出某个pid进程打开的文件

系统定时任务

  • crond可用于服务管理与访问控制
    • 一般是启动状态且设置为自启动
    • services crond restart，利用services启动或重启
  • 用户的crontab设置
    • 使用前先启动crond服务
    • crontab + 选项
      • -e：编辑crontab定时任务
      • -l：查询crontab任务
      • -r：删除当前用户所有的crontab任务（若要删除一个，按e编辑
    • -e，进入crontab编辑界面
      • 格式：xxxxx+所执行任务
      • 第一个x：一小时中第几分钟（0-59
      • 第二个x：一天中第几小时（0-23
      • 第三个x：一个月中第几天（1-31
      • 第四个x：一年中第几个月（1-12
      • 第五个x：一周中第几天（0-7，0、7都表示周日
      • 特殊符号
        • ：表任何时间，如第一个为表一小时中每分钟都执行一次
        • ，：表不连续时间，如第二个的8,12,16 表每天的八点、12点、16点执行一次命令
        • -：代表连续的时间范围，如最后的1-6表示周一到周六
        • /n：表每隔多久执行一次，如第一个的/10表示每隔十分钟执行一遍
      • 其它
        • 45 22 * + 命令：22点45分执行命令（每天
        • 星期几和每月的第几天最好不要同时出现，易混淆
        • 最小的识别范围为分钟
    • 举例说明
      • 定时关机或重启
      • 定义执行某自定义脚本，如备份脚本
