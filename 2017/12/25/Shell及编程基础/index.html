Shell及编程基础

概述

  • 什么是Shell
    • 1，是一个命令解释器，连接用户与系统内核
    • 由内而外：硬件-系统内核-Shell-应用程序-用户
    • 2，是一个功能强大的、解释执行的编程语言，可直接调用Linux系统命令
  • 分类
    • Bourne Shell：从1979起Unix就开始使用，其主文件名为sh
    • C Shell：主要在BSD版的Unix使用，其语法类似C语言而得名
    • Bash：与sh兼容，如今Linux的基本shell便是Bash
    • Shell两种主要语法类型：Bourne、C
      • 两种语法彼此不兼容
      • Bourne家族：sh、ksh、Bash、psh、zsh（Bash为主流
      • C家族：csh、tcsh
  • Linux支持的Shell
    • vim /etc/shells
    • 有：sh、bash、nologin、csh、tcsh

shell脚本的执行方式

  • echo输出命令
    • echo + 选项 + 输出内容
    • -e：支持反斜线控制的字符转换
    • 如echo “ab/bc”：输出ac，/b退格，即向左删除
  • 编写测试脚本
    • vim hello.sh
    • 第一行：#! /bin/Bash（表示此脚本为shell脚本，而非注释，不可省略
    • 第二：echo “hello world ”
  • 脚本执行
    • 赋予执行权限，直接运行（常用
      • chmod 755 hello.sh
      • ./hello.sh（或绝对路径
    • 通过Bash调用
      • bash hello.sh
      • 甚至不用赋予执行权限

Bash的基本功能

  • 历史命令
    • history + [选项]+[历史命令保存文件]
      • 直接history：显示历史命令
        • 默认保存1000条
        • 可在环境变量配置文件/etc/profile中修改（HISTSIZE字段
        • 其配置文件生效需要用户重新登录
      • -c：清空历史命令
        • 缓存中和文件中都会清空（ubuntu中测试时，并不会清空文件里的，只会清空缓存）
        • 不建议使用，但攻击者常用来擦除痕迹
      • -w：把缓存中历史命令写入历史命令保存文件
        • 手动执行命令来写入，默认用户退出后，会自动写入
        • 默认为～/.bash_history，也可手工指定
    • 历史命令的保存
      • 默认保存1000条
      • 可在环境变量配置文件/etc/profile中修改（HISTSIZE字段
      • 其配置文件生效需要用户重新登录
      • 默认保存在～/.bash_history
    • 历史命令的调用
      • 上下箭头显示上一条、下一条命令
      • !n：再次执行第n条命令
      • !!：再次执行上一条命令
      • !字符串：再次执行最后一条、以该字符串开头的命令
  • 命令别名及补全
    • 常用命令
      • alias 别名=‘原命令’：为命令设置别名（如alias vi=‘ vim’
      • alias：查询已有的命令别名
      • unalias + 别名：删除别名
    • 命令执行时顺序
      • 第一顺位：绝对路径或相对路径执行的命令
      • 第二顺位：执行别名
      • 第三顺位：执行Bash内部命令
      • 第四顺位：执行按照$PATH环境变量定义的外部命令
    • 其它
      • 别名不可与已有系统命令相同，防混淆
      • 让别名永久生效：vi /root/.bashrc后再添加目录（不同用户，家目录不同
    • 命令或文件的补全
      • Tab键自动补全
  • Bash常用快捷键
    • ctrl + 字母
    • a/e：光标移动到命令行开头/结尾
    • l：清屏，同clear命令
    • c：强制终止当前命令
    • u/k：删除或剪切光标之前/之后的内容（避免退格键一个一个删；粘贴则是剪切，不粘则是删除
    • y：粘贴u或k剪切的内容
    • r：在历史命令中搜索
    • d：退出当前终端，即退出登录，同logout
    • z：暂停，并放入后台
    • s/q：暂停/恢复屏幕输出
  • 输入或输出重定向
    • 标准输入输出
      • 键盘
        • 设备文件名：/dev/stdin
        • 文件描述符：0
        • 类型：标准输入
      • 显示器
        • 设备文件名：/dev/stdout
          • 文件描述符：1
          • 类型：标准输出
        • 设备文件名：/dev/stderr
          • 文件描述符：2
          • 类型：标准错误输出
    • 输出重定向
      • 其它
        • 命令执行结果原本输出到屏幕，改变其方向，输出到文件或设备中
        • 有输出的命令才可以用于重定向
        • 错误输出要在大于号前加2，且不可有空格
        • 正确输出和错误输出来分别保存用处不大，需同时保存
      • 正确输出和错误输出不同时保存
        • 标准输出重定向
          • 以xx的方式，把命令的正确输出输出到指定文件或设备中
          • 命令 > 文件：以覆盖的方式
          • 命令 >> 文件：以追加的方式
        • 标准错误输出重定向
          • 以xx的方式，把命令的错误输出输出到指定文件或设备中
          • 错误命令 2> 文件：以覆盖的方式
          • 错误命令 2>> 文件：以追加的方式
        • 正确输出和错误输出来分别保存用处不大，需同时保存，即不管正确与否，都保存
      • 正确输出和错误输出同时保存（建议
        • 正确输出和错误输出写入同一个文件
          • 不管正确与否都写入同一个文件
          • 方式1
            • 命令 > 文件 2>&1（覆盖
            • 命令 >> 文件 2>&1（追加
            • 后面固定是：2>&1
          • 方式2
            • 命令 &> 文件（覆盖
            • 命令 &>> 文件（追加
            • 推荐使用，易记
        • 正确输出追加至文件1，错误输出追加至文件2
          • 命令 >> 文件1 2 >> 文件2
    • 输入重定向
      • 命令执行的参数原来是键盘输入，更改为从文件输入
      • 用的不如输出重定向多，以wc为例测试
      • wc+[选项] +[文件名]
        • 直接wc：输入字符，ctrl+d结束，会显示输入的行数、单词数、字节数
        • -c：统计字节数
        • -w：统计单词数
        • -l：统计行数
      • 命令 < 文件
        • 把文件作为命令输入，可统计文件内容相关信息
        • 如wc < test.txt
      • 命令 << 字符串
        • wc << hello，一直输入，直到遇到hello作为结束，取之间内容作为参数（用的更少，了解
  • 多命令顺序执行
    • 分号;
      • 命令1；命令2
      • 多个命令顺序执行
      • 命令之间无逻辑关系
      • 如：date: dd命令 ;date（查看执行dd命令所花费时间
    • &&
      • 概述
        • 命令1 && 命令2
        • 逻辑与
      • 执行逻辑
        • 1正确执行，2才会执行
        • 1执行不正确，2不会执行
      • 应用
        • 用于有严格顺序要求的多个命令间，前面执行成功才可以执行后者，前面出一点错就全部停止
        • 如安装软件时：./configure && make && make install
    • ||
      • 概述
        • 命令1 || 命令2
        • 逻辑或
      • 执行逻辑
        • 1执行不正确，2才会执行
        • 1正确执行，2不会执行
      • 应用
        • 同逻辑与结合，判断某命令是否执行成功
        • 如：命令 && echo yes || echo no（很巧妙，多用于脚本内判断是否执行成功，if语句作用
    • 其他
      • dd命令
        • dd if=输入文件 of=输出文件 bs=字节数 count=个数
        • if：指定源文件或源设备
        • of：指定目标…
        • bs：指定一次输入/输出多少字节，即把多少字节看作一个数据块
        • count：指定输入/输出多少个数据块
        • 特殊的复制命令：磁盘、数据、分区、硬盘等的复制，不同于cp文件复制
  • 管道符（“|”）
    • 简介
      • 命令1 | 命令2
      • 1的正确输出作为2的操作对象
      • 前提是1必须成功执行，否则2不可执行
    • 举例
      • ls -a /etc/ | more
      • more原来跟文件，分屏显示其内容，现在操作对象变为前者命令的输出
      • netstat -an | grep “ESTABLISTED”
      • 查看当前系统的远程连接
    • 其它
      • grep命令
        • grep + 选项+“搜索内容” + 文件名
        • 显示其所在的行
        • -i：忽略大小写
        • -n：输出行号
        • -v：反向查找，如-v root，会排除root的那一行
        • —color=搜索出的关键字用颜色显示
  • 通配符及其它特殊符号
    • Linux常见通配符
      • ?：匹配一个任意字符
      • *：匹配0个或多个，即任何内容
      • []：匹配中括号中任意一个字符
        • [abc]表一定匹配一个，a或b或c
      • [-]：匹配其中任意一个字符，-表范围
        • [a-z]：表匹配一个小写字母
      • [^]：逻辑非，匹配不是中括号中的字符
        • [^0-9]：匹配一个不是数字的字符
    • 通配符应用
      • rm -rf *：删除所有文件
      • ls ?abc：列出所有abc前一个字符的文件
      • ls [0-9]*：列出数字开头的文件
      • ls [^0-9]*：列出非数字开头的文件
    • 其它特殊符号
      • 单引号
        • 单引号中的特殊符号都失去各自特殊意义
        • 引起含空格的字符串，使其为一个整体（Linux中空格作为命令、选项、参数的分隔
      • 双引号
        • 双引号中的特殊符号会失去各自特殊意义
        • 仍有特殊意义的
          • $：调用变量的值
          • 单个反引号：引用命令
          • \：转义字符
      • 反引号
        • 其括起来的内容是系统命令
        • Bash中会优先执行，然后再赋值等，如n=‘date’，先执行date命令，将其结果赋值给n变量（输入法问题，故用单引号代替
        • 同$()，但其易看错，不建议使用
      • $()
        • 同反引号，引用系统命令
        • 优先执行，如echo $(ls)，会输出ls执行结果
        • 推荐使用
      • #
        • shell脚本中作注释符
        • 配置文件中的注释符
      • $
        • 用于调用变量的值，如$name
      • \
        • 转义，其后的特殊字符失去特殊意义，变为普通字符

Bash的变量

  • 概述
    • 变量设置规则
      • 由字母、数字、下划线组成，不可数字开头
      • Bash中变量默认类型都是字符串型，若要进行数值运算，则必须指定变量类型为数值型
      • 变量用等号连接值，等号左右两侧不能有空格
      • 变量的值若有空格，用单/双引号括起来
      • 变量的值，允许有转义符\
      • 若要增加变量值，可进行变量值叠加。不过变量需要用双引号包含”$变量名”或者${变量名}包含
      • 若把命令的输出结果作为值赋值给变量，需要用反引号或$()包含名令
      • 环境变量名建议大写，便于与命令区分
    • 变量分类
      • 用户自定义变量
        • 最普通、最常见的
        • 变量名及变量名均自定义
      • 环境变量
        • 此变量保存的是与系统操作环境相关的数据
        • 系统环境变量不可更改
        • 用户可自行新建新的环境变量
      • 位置参数变量
        • 用来向脚本中传递参数和数据
        • 变量名不能自定义
        • 变量作用是固定的
        • 从根本上说，就是预定义变量，只不过出现太频繁
      • 预定义变量
        • Bash中已定义好的
        • 变量名不能自定义
        • 变量作用是固定的
      • 自上往下：限制越来越严格，后两者限制程度相同
  • 用户自定义变量（即本地变量
    • 变量定义
      • 如name=“test test”（有空格，加引号
      • echo $name（调用时加$
    • 变量叠加
      • a=123
      • a=“$a”456（此时为123456
      • a=${a}789（此时为123456789
    • 变量调用
      • echo $name（调用时加$
    • 变量查看
      • set命令
      • 显示系统所有变量（不止自定义的
    • 变量删除
      • unset命令
      • unset name（不需加$
  • 环境变量
    • 何为环境变量
      • 用户自定义变量只在当前shell有效
      • 环境变量会在当前shell和此变量所有的子shell中有效
      • 若把环境变量写入相关配置文件，则其会在所有的shell中生效
      • 类似于局部变量和全局变量作用
    • 创建子shell
      • 当前shell为bash，再键入bash，会进入子shell
      • pstree可以查看
      • exit退出当前子shell，回到父shell
    • 设置新的自定义环境变量
      • export 变量名=变量值
        • 声名为环境变量
        • 或者，变量名=变量值，再export 变量名
        • 加$调用变量，如echo $name
      • env
        • 查询变量
      • unset 变量名
        • 删除变量
        • 如unset name（不需$
    • 常见系统环境变量
      • PATH
        • 作用：定义系统查找命令的路径
        • echo $PATH：输出许多路径，冒号分割
        • 所有的执行都需要到绝对路径，ls等系统命令也不例外
        • 因PATH的存在，ls可不加绝对路径，直接运行
        • 可变量叠加，将自习脚本路径写入path，但仅本次生效，重启失效
      • PS1
        • 作用：定义系统提示符的变量
        • 并非严格意义上的系统环境变量，env不可见，set可见
        • 默认值（echo $PS1后）：[\u@\h \W]\$
        • 默认提示符是[root@localhost ~]#（即每个shell窗口前的提示信息，
        • 可自定义，但意义不大，默认即可
  • 位置参数变量
    • $n
      • n为数字，$1–$9表第一到第九个参数
      • $0表示命令本身
      • 10以上的需要用大括号包含，如${10}
      • 运行脚本文件时：./test.sh 11 22 33，则分别对应$0-3，此参数可以在脚本内容中调用
    • $*
      • 代表命令行中所有参数
      • $*把所有参数看作一个整体
    • $@
      • 也代表命令行中所有参数
      • 只不过把每个参数区别对待，可理解为一个数组有多个元素
    • $#
      • 代表命令行中所有参数的个数（不包括命令本身
  • 预定义变量
    • 变量名和变量值均不可改，只可传入不同的值
    • 常用预定义变量
      • $?
        • 最后一次执行的、命令的返回状态
        • 若此变量值为0，表上一命令成功执行
        • 若非0，表执行不正确（具体哪个值由命令决定，如命令不存在时爆127，文件未找到爆2
      • $$
        • 当前进程的进程号PID
      • $!
        • 后台运行的最后一个进程的进程号
    • 接受键盘输入
      • read +选项+变量名
        • shell脚本中使用，类似py中raw_input()
        • -p + 提示信息：在等待read输入时，输出提示信息
        • -t+秒数：不加read会一直等待，加了会指定等待的时间
        • -n + 字符数：接受到指定的字符数后就会执行，就不用手动按回车
        • -s：隐藏输入的数据，适用于机密信息（如Linux输入密码时
  • 数值运算与运算符
    • declare声名变量类型
      • declare [+/-] 选项 变量名
      • -：给变量设定类型属性（与日常思维相反
      • +：取消变量的类型属性
      • -i：将变量声明为整数型（integer
      • -x：将变量声明为环境变量（export也可以
      • -p：显示指定变量的被声名的类型（如环境变量显示x
    • 数值运算
      • 方法1（使用declare
        • aa=11
        • bb=22（给aa和bb赋值
        • declare -i cc=$aa+$bb
          • 此时cc为33，一般的话是字符串”11+22”
          • 当然也可以为其他运算符
      • 方法2（使用expr或let数值运算工具
        • aa=11
        • bb=22（给aa和bb赋值
        • dd=$(expr $aa + $bb)
          • dd为33
          • 注意加号两侧必须有空格
      • （推荐）方法三：$((运算式))或$[运算式]
        • aa=11
        • bb=22（给aa和bb赋值
        • ff=$(($aa+$bb))
        • 或gg=$[$aa+$bb]
    • 常见运算符
      • 各运算符存在优先级
      • 优先级数目越大，其优先级别越高
      • 可用小括号调整优先级
  • 变量测试与内容替换
    • 随用随查，知道这回事即可
    • 举例如x=${y-新值}
      • 若变量y没有设置，则x=新值
      • 若y为空值，则x为空
      • 若变量y设置值，则x=$y
      • 上述逻辑条件反之亦可成立
      • 不推荐记忆使用，了解即可（有相应表格可查看

环境变量配置文件

  • 简介
    • 定义系统默认环境变量，如PATH，PS1，HISTSIZE，保存在此的就会永久生效
    • source命令
      • source+配置文件或 .+配置文件
      • 默认下需要重启系统才可让环境变量配置文件生效，source即可避免重启
    • 主要环境变量配置文件
      • /etc/profile
      • /etc/profile.d/*.sh
      • /etc/bashrc
      • ～/.bash_profile
      • ～/.bashrc
  • 文件作用
    • etc下的针对所有用户，家目录下的只针对当前用户
    • 主要环境变量配置文件
      • /etc/profile
        • USER变量
        • LOGNAME变量
        • MAIL变量
        • PATH变量
        • HOSTNAME变量
        • HISTSIZE变量
        • umask:
        • 调用/etc/profile.d/*.sh
      • /etc/profile.d/*.sh
      • /etc/bashrc
      • ～/.bash_profile
      • ～/.bashrc
  • 其他配置文件和登录信息
    • ～/.bash_logout（注销时生效的环境变量配置文件
      • 默认为空
    • ～/bash_history
      • 保存历史命令
    • shell登录信息
      • 本地终端前欢迎信息
        • /etc/issue
      • 远程登录前欢迎信息
        • /etc/issue.net
        • 转义符在其中不可使用
        • 是否显示此欢迎信息，由ssh的配置文件/etc/ssh/sshd_config决定，加入“Banner /etc/issue.net”才可
      • 登录后欢迎信息
        • /etc/motd
        • 本地登录还是远程登录时都会显示
        • 是登录后的

正则表达式

  • 正则与通配符
    • 其他语言中，通配符只是正则内容的一部分
    • 正则表达式
      • 在文件中匹配符合要求的字符串
      • 包含匹配
        • 类似于贪婪模式，尽可能多的匹配
        • 如grep “abc” test.txt，把所有123所在行显示出来
      • grep、awk、sed等搜索文件内容的命令支持正则
    • 通配符
      • 系统目录中匹配符合条件的文件名
      • 完全匹配
        • 如目录下存在aa和aabb两文件，ls aa只会显示aa文件；ls aa*才会全部显示
      • ls、find、cp等与文件名相关的命令，不支持正则，故只可使用通配符
  • 基础正则表达式（除基础还有扩展
    • 元字符
      • *
        • 前一个字符匹配0次或任意多次
        • 0次的也会
        • 前一个字符
        • 通配符*表任意字符
      • .
        • 匹配除了换行符外任意一个字符
        • 类似通配符中？
      • ^
        • 匹配行首
        • ^hello，匹配hello开头的
      • $
        • 匹配行尾
        • hello$，匹配hello结尾的
      • []
        • 匹配其中指定的任意一个字符，仅一个
        • 同通配符中｛｝
        • [0-9]，匹配一个数字
      • [^]
        • 匹配除其中指定的 任意一个字符，同上恰相反
        • [0-9]，匹配一个非数字
      • \
        • 转义符，取消特殊字符的特殊含义
      • {n}
        • 匹配其前面的字符恰好出现n次
        • 如[0-9]{4}，匹配四个数字
        • \转义大括号｛
      • {n,}
        • 匹配其前面的字符出现大于n次
        • 如[0-9]{4,}，匹配四位以上数字
      • {n,m}
        • 匹配其前面的字符出现大于n次，小于m次
        • 如[0-9]{4,8}，匹配4-8位数字
    • 举例
      • grep “.*” test.txt，匹配所有内容

字符截取命令

  • cut列提取命令
    • cut +选项+文件名
      • 可以手工指定文件名，也可通过管道符
      • -f+列号：提取指定列，逗号分隔指定多列
      • -d+分隔符：按照指定的分隔符分隔列
    • 举例
      • cut -f 2,4 test.txt ，提取出24列
      • cut -d “:” -f 1 /etc/passwd，提取passwd文件第一列，即用户名列，其文件每行以冒号分隔
    • 独立使用意义不大，多与grep搭配
      • grep是提取行，cut提取列
      • cat /etc/passwd | grep /bin/bash | grep -v root | cut -d “:” -f 1
      • 提取出所有可登录用户，shell为/bin/bash的为系统可登录用户
      • 排除root，grep的v选项
    • cut命令的局限
      • 文件是一个x行x列的表格，制表符分隔，不可空格，否则cut不可使用
      • 不可识别以空格分隔的，易报错
      • 可识别：制表符或冒号等符号
      • awk解决此局限，但awk较复杂，简单的能cut就cut，如冒号分隔的
  • printf标准格式输出命令
    • printf +“输出类型及格式” +输出内容
      • 输出类型
        • %ns
          • 输出字符串
          • n为数字表示输出几个字符
        • %ni
          • 输出整数
          • n为数字表示输出几个数字
        • %m.nf
          • 输出浮点数
          • m为总位数，n为数位
          • %8.2f表总共输出8位，2位小数，6位整数
      • 输出格式
        • \a：发出警告音
        • \b：输出退格，及backspace
        • \f：清屏
        • \n：换行
        • \r：回车，即enter
        • \t：水平输出退格，即tab
        • \v：垂直输出退格，即tab
    • 其它
      • 严格来讲并不是字符提取命令，包含于awk
      • 其后不可直接加文件名，其前也不可以有管道符
        • 利用$()，即先执行其中的命令，其输出结果作为printf的输出内容
        • 如printf ‘%s’ $(cat test.txt)
      • 不如echo命令方便，也不如cat，cat命令本身做了格式优化
      • 为awk命令使用方便，因awk不可调用echo和cat
      • awk中支持print和printf
        • print在每次输出后自动加换行符，print并不是Linux中的命令
        • printf需手工加入换行符
    • 举例
      • printf ‘%s %s %s\n’ 1 2 3 4 5 6
      • 分两组，123和456，且换行了
  • awk命令
    • 简介
      • 同cut截取列，但会弥补其缺陷；grep是截取行
      • awk复杂且功能强大，故有awk编程一说
    • awk ‘条件1 ｛动作1｝条件2 ｛动作2｝…’ 文件名
      • 单引号扩起来
      • 后可加文件名，前可加管道符
      • 满足条件则执行动作，不加条件时表示总会执行动作
      • 以行为单位先遍历，再根据条件是否符合决定动作的执行与否（自己总结，待验证
      • 条件（pattern
        • 一般使用关系表达式作为条件
        • x > 10，判断x是否大于10
        • x >= 10，
        • x <= 10，
      • 动作（action
        • 格式化输出
        • 流程控制语句
    • 举例
      • awk ‘｛printf $2 “\t” $6 “\n”｝’ test.txt
        • printf需要手动换行
        • $2表第二列
      • df -h | awk ‘｛print $1 “\t” $3｝‘
        • 磁盘使用情况
        • 不加条件时则默认执行动作
      • awk ‘$2>=20｛printf $2 “\n”｝’ test.txt
        • $2表第二列
        • 第二列数据大于20时才执行
    • BEGIN和END
      • 处理数据之前或之后执行某动作，属于条件
      • BEGIN常与FS一起使用，指定分隔符时
    • FS内置变量
      • 默认识别的分隔符是制表符或空格，FS可指定
      • awk ‘BEGIN｛FS=“:”｝｛printf $2 “\t” $6 “\n”｝’ /etc/passwd
      • 以冒号为分隔，打印出其第26行
      • 要加BEGIN，否则第一条数据不会认为以冒号分隔的
  • sed命令
    • 简介
      • 并非截取命令，grep行截取，awk和cut列截取
      • 而是一个流编辑器，将数据进行选取、替换、删除、新增
      • vim也是编辑器，操作文件内容，不可操作命令输出结果，sed两者都可以（借助管道符
    • sed +选项+‘动作’+文件名
      • 动作用单引号扩起来
      • 选项
        • -n
          • sed默认将所有数据都输出到屏幕
          • 加此选项，会把经过sed处理后的输出到屏幕
        • -e
          • 允许对输入数据应用多条sed命令编辑
          • 如，sed -e ‘3a yyy’ ‘2a hello’ test.txt
        • -i
          • 默认sed处理后不会影响源文件，只是输出时有所改变
          • 加此i，sed处理后的结果写入源文件，直接修改了源文件，而不是输出至屏幕
          • 存在一定危险性，不建议，sed还是多用于命令结果处理
      • 动作
        • 一般之前要紧跟行或行范围，如2a，3i
        • a \
          • 追加，在当前行后添加一行或多行
          • 添加多行时，除最后一行，每行末都加\表示未完结
        • c \
          • 行替换，用c后面的字符串替换原数据行
          • 替换多行时，除最后一行，每行末都加\表示未完结
        • i \
          • 插入，在当前行前插入一行或多行
          • 插入多行时，除最后一行，每行末都加\表示未完结
        • d
          • 删除指定行
        • p
          • 输出制定行
        • s
          • 字符串替换
          • 行范围s/旧/新/g（同vim
    • 举例
      • sed -n ‘2p’ test.txt
        • 输出第二行
        • 若不加n，在所有数据基础上又打印第二行，第二行重复输出
        • 一般打印时，都跟n
      • df -h | sed -n ‘2p’
        • 管道符后以处理上条命令结果
      • sed ‘2,4d’ test.txt
        • 删除2-4行，但不修改文件本身
      • sed ‘2a hello’ test.txt
        • 第二行后追加hello
      • sed ‘2i hello \ world‘ test.txt
        • 第二行前插入两行数据
        • -i时，直接把修改写入文件
      • sed ‘2c hello world‘ test.txt
        • 行替换，将第二行替换为指定的字符串
      • 字符串替换
        • sed ‘s/旧/新/g’ 文件名
        • sed ‘3s/yes/no/g’ test.txt
        • 在第三行，将yes替换为no

字符处理命令

  • 排序命令sort
    • sort+选项+文件名
      • -f：忽略大小写
      • -n：以数值型进行排序，默认是字符串型
      • -r：反向排序
      • -t：指定分隔符，默认是制表符
      • -k n[,m]：按照指定的字段范围排序，n开始m结束；默认到行尾
    • 按每行的第一个字母来排序
    • 举例
      • sort /etc/passwd
      • sort -t “:” -k 3,3 /etc/passwd（3开头3结尾，即用3字段排序
  • 统计命令wc
    • wc+选项+文件名
      • -l：只统计行数
      • -w：只统计单词数
      • -m：只统计字符数
      • 不加则输出所有，行数，单词数，字符数
    • 也可以用管道符来接收

条件判断

  • 按文件类型进行判断
    • -e：只判断文件是否存在
    • -s：判断文件是否存在，并且是否为空
    • -x + 文件：判读该文件是否存在，并且是xx文件（是的话返回真
      • -d：目录文件
      • -f：普通文件
      • -b：块设备文件
      • -c：字符设备文件
      • -L：符号链接文件
      • -p：管道文件
      • -S：套接字文件
  • 按照文件权限判断
    • -xx +文件：判断该文件是否存在，并且是否存在xx权限，（存在xx权限则返回真
    • -r/w/x：读写执行权限（ugo其中有一个存在此权限就返回真
    • -u：SUID权限
    • -g：SGID权限
    • -k：Sbit权限
  • 两个文件之间进行比较
    • 文件1 -nt 文件2:文件1的修改时间是否比2新
    • 文件1 -ot 文件2: 文件1的修改时间是否比2旧
    • 文件1 -ef 文件2
      • 判断文件1和2的Inode号是否一致，即是否是同一个文件；
      • 用于判断硬链接
  • 两个整数之间比较
    • 整数1 -xx 整数2: 1和2是否xx，若xx则为真
    • -eq：是否相等
    • -ne：是否不相等
    • -gt：是否大于
    • -lt：是否小于
    • -ge：是否大于等于
    • -le：是否小于等于
  • 字符串的判断
    • -z 字符串
      • 字符串是否为空，为空返回真
    • -n 字符串
      • 是否为非空
    • 字符串a == 字符串b
      • 是否相等
    • 字符串a != 字符串b
      • 是否不相等
  • 多重条件判断
    • 判断1 -a 判断2
      • 逻辑与
      • 12都真才真
    • 判断1 -o 判断2
      • 逻辑或
      • 12有一个为真便真
    • ！判断
      • 逻辑非
      • 对原判断取反
  • 两种判断格式
    • 利用test
      • test -e /root/install.log
      • 运行后无输出，借用echo $?，来判断上一条是否执行正确，若是0则正确，非0则错误
    • 利用中括号
      • 中括号右/左必须有括号
      • [ -e /root/install.log ]
      • [ 23 -gt 22 ]
    • 两种方式屏幕都无输出
      • 运行后无输出，借用echo $?，来判断上一条是否执行正确，若是0则正确，非0则错误
      • [ -e /root/install.log ] && echo yes || echo no
