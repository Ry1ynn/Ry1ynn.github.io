1-计算机基础知识

半导体存储器：

  • 所说的内存就是这种，但还有其他
  • 可分为：随机存取存储器（RAM）、只读存储器（ROM）
  • ROM：只读存储器
    • 只能读出不能随意写入信息
    • 主板上的ROM里面固化了一个基本输入/输出系统，称为BIOS
    • 是一种只能读取资料的内存,断电后不丢失信息
  • RAM：随机存取存储器
    • 高速存取，任意位置读写时间相等(即与地址无关)
    • 通常用来加载操作系统、各种正在运行的软件
    • 常说的内存主要是指RAM。
  • 速度快，体积小

存储器分为：内存、外寸

  • 内存分为：RAM（主存）、ROM
  • 外存：硬盘、磁盘等
  • 主存 != 内存，内存 > 主存
  • 容量的单位为字节数

进制转换问题

别想的太繁杂，记住主要原则，学会类比

  • X转2进制
    • 10转2进制
      • 法1: 整数除2逆向取余，小数乘2正向取整（小数时不好结束，取几位即可
      • 法2: 自己想的8421-3210法，如13=8+4+1=3+2+0，即1101
    • 8转2进制：8的每位用2的3位表示
    • 16转2进制：16的每位用2的4位表示
  • 2进制转X
    • 2转10：自己的8421-3210法，熟记2的n次幂，1、2、4、8、16、32、64、128（X转10也是此思想）
    • 2转8：每3位一组
    • 2转16：每4位一组
  • 因此：可将2进制看作中介

数据与指令

  • 计算机由输入设备，输出设备，运算器，存储器和控制器五大部件组成
  • 指令
    • 是计算机能够识别的规定好含义的一系列数字
    • CPU能够读取指令，并按照设计好的电路，执行指令含义
    • 支配计算机中的信息传递，是构成计算机软件的基本元素
  • 数据是计算机中被计算或者显示的数字，也是构成软件的基本元素

  • 指令和数据从存储的角度看没有本质的不同，指令就是计算机能够识别具有特定含义的数据

    • 本质上是相同的，都以二进制的形式存放在计算机中；

    • 二者功能不同：指令用于指导计算机去做什么以及怎样做，而数据是计算机操作的具体对象；

    • 也可以根据访的时机来区分：二者都存储在存储器中，在取指令周期内取出的是指令，同理，在读数据周期内取出的为数据。

编程语言分类

  • 主要分为三大类：机器语言，汇编语言，高级语言
  • 机器语言
    • 为纯粹的机器代码，由010101……的二进制代码组成
    • 可以由计算机直接执行，运行效率最高
    • 通用性不强，对于不同的硬件需要不同的程序
  • 汇编语言
    • 可分为8086汇编，arm汇编等
    • 运行效率低于机器语言（但也很快了）
    • 用简单的助记符代替二进制代码
    • 通用性不强，对于不同的硬件需要不同的程序

  • 高级语言

    • 有Java、C、C++、python、php等，用人类可识别的自然语言（主要英语）进行编程
    • 运行效率更低，但编程难度和程序可读性大幅提高
    • 可移植性好，实用性较好，适合大规模开发，是现在大多数程序员选择的语言

  • 在面向机器或面向用户的角度，可分为三大类，即机器语言、汇编语言以及高级语言；

  • 高级语言中有C/C++、Java、python、PHP 等，若有需要，也可进行细分，比如 CC++更面向底层，而 PHP 或 python 等对底层进行了进一步的封装，可以说更加面向用户。
  • 在程序执行的角度，可分为编译型语言和解释型语言。

编译型与解释型语言

  • 为何要翻译：
    • 一般编程都是用的高级语言(写汇编和机器语言的大牛们除外)
    • 计算机不能直接理解高级语言，只能理解和运行机器语言
    • 所以必须要把高级语言翻译成机器语言，计算机才能运行高级语言所编写的程序
  • 翻译的方式有两种
    • 一个是编译，一个是解释
    • 两种方式只是翻译的时间不同
  • 编译型语言
    • 用编译型语言写的程序执行之前，需要一个专门的编译过程，把高级语言翻译成机器语言（比如windows下的exe文件
    • 以后就可以直接运行而不需要编译了，因为翻译只做了一次，运行时不需要翻译
    • 所以编译型语言的程序执行效率高

  • 解释型语言

    • 解释则不同，解释型语言编写的程序不需要编译
    • 解释型语言在运行的时候才翻译，比如python语言
    • 在执行的时候，专门有一个解释器能够将python语言翻译成机器语言
    • 每个语句都是执行的时候才翻译
    • 这样解释型语言每执行一次就要翻译一次，效率相对比较低

  • 编译型语言：在程序执行之前，源代码就翻译成机器指令，由于事先翻译了，所有执行时速度快，如 C/C++

  • 解释性语言：程序运行之前，不会事先翻译成机器指令，而是一边翻译一边执行，因此，执行速度相对较慢，如 python

2-计算机中的数

字节与字长

  • 字长：CPU 一次可处理的数据的最大长度，其位数取決于具体的计算机
  • 字节：衡量数据量以及存储容量的基本单位，1 字节等于 8 位二进制信息

表示范围

  • 为何n位有符号数的范围负数比正数多一个？如：
    • 8位有符号数的取值范围是-128 ~ +127
    • 16位有符号数的取值范围是-32768 ~ +32767
  • n位有符号数表示范围：-2^(n-1) ~ 2^(n-1)-1，暂不深究原理（意义不大且麻烦
  • n位无符号数表示范围：0 ~ 2^n - 1，如8位无符号范围0～255

7-一维数组

  • 数组长度、下标

    • 定义时：长度只可为常量
      • 调用时：下标可常量，可变量

  • 数组定义、存储方式

    • 数组是类型相同，数目固定的若干个变量的有限集合，用统一的数组名和下标来管理数组中的元素
    • 存储时，占用连续的若干个存储单元，数组名就是这若干个存储单元的起始地址，所以数组名的地址是不能改变的

  • 数组定义格式：类型 数组名[元素个数]

  • 初始化方法

    • 在定义数组时，对其整体初始化

    • 对于字符数组的初始化

    • 定义完数组后，只能一个ー个的对每个元素赋值，不能像定义一样整体賦值

    • 数组初始化

      • 整体赋值：a[3] = {1,2,3}
      • 部分赋值：a[3] = {1,2}（剩余元素补0）
      • 灵活赋值：a[] = {1,2,3}（不给长度，根据给的值灵活分配）

    • 数组初始化与赋值

      • 初始化：定义时就给值，称初始化
      • 赋值：定义之后才给值，称赋值（赋值名词的使用无需太严格）

11-选择结构switch

switch-case基本

  • 执行逻辑：

    • 执行 switchi 中的表达式，与 case 的常量进行匹配，如果相同执行 case 后的语句；
    • 一般在 case 语句添加 break；
    • 如果没有配到 case 语句，最后会执行 default 的语句

  • 只能基于整数值进行，而不能是浮点型（字符也可以，因为其本质上也是数字，整型数字）

  • default与case的顺序无关紧要，不影响结果
  • switch结构能实现的，if结构也能实现，反之不然

适应情况

  • switch：分支结构简单，无规律分散且条件为整数时使用

  • if else: if-else 语句更适合于对区间（范围）的判断

  • if结构适应情况

    • 逻辑复杂
    • 分支较小
  • switch结构适应情况
    • 逻辑简单
    • 分支较多

优缺点

  • switch缺点
    • 只能基于整数值来进行匹配，不可浮点数
    • 只能判断是否相等，不可基于某区间范围来判断
    • switch可实现的if都能实现，反之不然，即存在一定限制
  • if缺点
    • 分支较多时用if会显得很乱
    • 多层if嵌套也会使得程序变得复杂

其他

  • 程序中尽量不要出现大量重复代码

14-循环结构

for循环

  • for循环中三个表达式，for(表达式1, 表达式2, 表达式3)

    • 表达式1：循环变量赋值
    • 表达式2：判断循环条件
    • 表达式3：控制循环结束

  • for循环中表达式1、2、3与循环体的执行

    • 表达式1：首先执行，且只执行一次
    • 表达式2-循环体-表达式3：循环执行，直至结束

三个循环比较

  • for与while

    • 已知循环次数，用for
    • 未知，用while

  • for、while与do-while

    • do-while：至少执行一次
    • for、while：可能一次也不执行

  • do-while与while

    • while：先判断，再执行，可能一次都不执行
    • do-while：先执行一次，再判断，至少执行一次；

  • 三种循环原则上可以相互替代，但实际上也要根据需求来选择

  • 分别的执行流程

  • 如何选择

    • 当不明确此循环会循环多少次的时候，一般使用while循环，内部依靠 breaks退出循环
    • 如果有需要必须要先做一次循环再做条件判断的话，可以使用 do while

if以及循环结构

  • 可以只执行一条语句（此时不用大括号）
  • 若要执行多条，一定要大括号包起来
  • 否则，默认只执行其紧跟的那一句，后面的他管不着

continue、break

  • continue：

    • 跳过本次循环，并不会结束循环，跳过本次之后还要再看判断条件，看是否能进入下一次的
    • 结束的是本次循环，而不是终止整个循环的执行。

  • break：

    • 结束循环，判断条件就不用看了
    • 结束的是整个循环过程，将不再判断执行循环的条件是否成立。

二维数组初始化

  • 分行赋值：int arr[2][3] = { {1,2,3}, {4,5,6} }
  • 整体赋值：
    • int arr[2][3] = { 1,2,3,4,5,6 }
    • 此种形式下，行可省列不可省，int arr[][3] = { 1,2,3,4,5,6 }，即可表示每行三个元素，具体几行，自己分去
  • 部分赋值（不足的补0）
    • 分行形式：int arr[2][3] = { {1}, {4,5} }（可保证每行都有值）
    • 整体形式：int arr[2][3] = { 1,2,3 }（后面的行不一定有值）

C中随机数相关

  • 与rand和srand相关，都位于stdlib.h，二者一起使用

    • rand，产生随机数，int rand(void)
    • srand，设置随机数种子，供rand使用，void srand(unsigned int seed)

  • 若每次 seed 都设相同值，则rand() 所产生的随机数每次就会一样

  • srand( (unsigned int) (time(NULL) ) 可产生不同的随机数种子，因为时间是在变化的，总会不同

  • 举例

  • 产生随机数时的小技巧，若要限制随机数在x以内，只要rand() % x即可，取余后，只能小于x（取余操作NB！！）

  • 目前三个安全版函数：scanf_s、strcpy_s、strcat_s,前者长度在第三个参数处，后两者在第二个位置；

开关变量（状态标示符

  • 程序设计中，开关变量的作用：

    • 开时才做某事，若某条件不符合了，立即关闭开关，关闭时不做
    • 条件符合——打开开关——执行操作
    • 开关初始化一个值（打开或关闭），遇到条件变化时，及时打开或关闭
