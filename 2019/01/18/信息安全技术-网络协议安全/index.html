信息安全技术-网络协议安全

发表于 2019-01-18 | 分类于 NISP二级

1.1-网络协议

  • 开放系统互联，ISO/OSI，七层
  • OSI安全架构：基于八类安全机制，提供五类安全服务

物理层

  • 电气、机械、规程特性；
  • 比特流转换为电压
  • 双绞线、同轴电缆、光纤
  • 中继器、集线器
  • 100BaseT等标准

数据链路层

  • 物理寻址（MAC地址），网络拓扑
  • 差错控制，检错和纠错
  • 封装成帧
  • 流量控制（滑动窗口
  • 网卡、网桥、交换机
  • PPP、HDLC、ARP等协议
  • 以太网协议标准，分两个子层
    • LLC：逻辑链路控制子层
      • IEEE 802.2
      • 为上层提供统一接口
      • 使上层独立于下层的物理介质
      • 提供流量控制、排序等服务
    • MAC：媒体访问控制子层
      • IEEE 802.3
      • 烧录到网卡ROM
      • 48b的MAC地址
      • 唯一性

网络层

  • 逻辑寻址（IP地址）、路由选择、分组交换
  • 三层交换机、路由器

传输层

  • 端到端、建立逻辑连接
  • 寻址基于端口号
  • UDP、TCP

会话层

  • 不同应用程序的数据隔离
  • 会话建立、维持、终止
  • 同步服务
  • 会话控制（单向或双向

表示层

  • 数据格式、协议、字符转换
  • 数据加解密、解压缩
  • ASCII、GIF、JPEG等格式

应用层

  • 应用的接口，面向用户
  • 网络访问流处理
  • 流量控制
  • 错误恢复
  • FTP、HTTP等协议

分层结构的优点

  • 降低复杂性（大问题切割成小问题，分散至各层
  • 促进标准化工作
  • 各层之间相互独立，一层的变化不会影响其他层
  • 协议开发模块化
  • 简化理解与学习

数据的封装与分用

  • 数据封装：数据发送时，从高到低，依次封装
  • 数据解封装：数据接收时，从低到高，依次解封装

TCP/IP与OSI的对应

  • 应用层：会话层、表示层、应用层
  • 传输层：传输层
  • 网际互联层：网络层
  • 网络借口层：物理层、数据链路层

1.2-网络接口层安全

网络接口层

  • 主要协议：ARP、RARP
  • 安全问题
    • 损坏：自然灾害、动物破坏、老化、误操作
    • 干扰：大功率电器、电源线路、电磁辐射
    • 电磁泄漏：传输线路电磁泄漏
    • 欺骗：ARP欺骗
    • 嗅探：常见二层协议是明文通信的（以太、ARP等
    • 拒绝服务：mac flooding、arp flooding

ARP欺骗

  • ARP相关：IP地址到mac地址、arp请求-广播、arp响应-单播、各主机存在映射表
  • 缺陷：对arp响应没有身份认证，即攻击者也可响应，这样在请求者的映射表中就记录了错误的记录，在以后的发送中，就发到了攻击者那，从而达到欺骗的目的
  • 欺骗大概过程：
    • a:1.1.1.1、b:2.2.2.2、c:3.3.3.3
    • a发arp请求，谁的IP是2.2.2.2？即问2.2.2.2的MAC地址是什么？
    • b和c都能收到
    • 正常情况，与其符合的才响应，即应该b响应
    • 但c出于欺骗目的，c对a响应，说我的IP是2.2.2.2，即2.2.2.2对应的Mac是b
    • 同时b对a说，1.1.1.1的mac是d
    • a找不到d，即不能与a正常通信（低级攻击，使ab不能通信
    • 若c告诉a：b的mac为c、告诉b：a的mac为c
    • c就成为二者中间人，中间人攻击（高级攻击）

ARP欺骗防御

  • 使用静态ARP缓存
  • IP与MAC地址绑定
  • ARP防御工具

注：PC端防御做的较好，而手机端薄弱，故少连公共Wi-Fi，信息很容易被中间人窃取

1.3-网络互联层安全

网络互联层

  • 主要协议：IP、ICMP、IGMP
  • IP是TCP/IP协议族中最核心的协议
  • IP协议特点：不可靠、无连接（传输层实现可靠传输
  • 安全问题：
    • 拒绝服务：分片攻击teardrop、死亡之ping
    • 欺骗：IP源地址欺骗
    • 窃听：嗅探
    • 伪造：IP数据包伪造

ping of death

  • ping-测试另一台主机是否可达
  • 查看网络上主机是否处于活动状态
  • ping原理：发送ICMP回显请求报文给目的主机，并等待ICMP回显应答，根据回显应答的内容来判断目的主机的状况
  • ping之所以会造成伤害是因为：早期操作系统在处理ICMP协议数据包时存在漏洞，如下：
  • ICMP协议的报文长度时固定的，大小为64KB，早期很多操作系统在接收ICMP数据报文的时候，只会开辟64KB的缓存区用于存放接收到的数据包
  • 一旦发送过来的ICMP数据包的实际尺寸超过64KB（65535B），操作系统将收到的数据报文向缓存区填写时，报文长度大于64KB，就会产生一个缓存溢出，结果将导致TCP/IP协议堆栈的崩溃，造成主机的重启动或死机
  • ping程序有一个“-l”参数可指定发送数据包的大小，故很容易实现此攻击，如：ping -l 65536 1.1.1.1
  • 若对方主机存在这样一个漏洞，就会形成一次拒绝服务攻击，成为死亡之ping
  • 如今操作系统都已做了修补，对可发送的数据包大小进行了限制，如攻击时，会显示：Bad value for option -l，valid range is from 0 to 65500

泪滴（Teardrop）

  • 泪滴也成为分片攻击，利用TCP/IP协议的问题，拒绝服务攻击，由于第一个实现此攻击的程序为teardrop，故也称泪滴
  • 数据包过大时，要发送端要进行分片，然后在接收端进行重组
  • 正常分片：0-9、9-13、13-19
  • 错误分片：0-9、6-13、13-19，第一、二个分片会有重叠，在接收端重组时会崩溃，故导致错误发生

Smurf攻击

  • 利用IP欺骗和ICMP回应包引起目标主机网络拥塞，实现Dos攻击
  • 攻击原理：
    • 在构造数据包时将源地址设置为被攻击主机的地址
    • 而将目的地址设为广播地址
    • 故，大量的ICMP echo 回应包被发送给被攻击主机，使其因网络拥塞而无法提供服务
    • 比ping of death洪水的流量高出1或2个数量级

1.4-传输层安全

TCP

  • 传输控制协议，面向连接，可靠的，字节流服务
  • 提供可靠性服务：确认、超时重传、流量控制等

UDP

  • 用户数据包协议，面向事务，不可靠
  • 特点：无连接、不可靠、协议简单、占用资源少、效率高

端口号

  • TCP/UDP通过16bit来识别应用程序
  • 知名端口号由internet号分配机构来管理
  • 现状
    • 1-255：知名的网络服务
    • 256-1024：unix操作系统特定的端口号
    • 1024-5000：临时分配的端口号
    • 5000以上：保留给应用程序

TCP建立连接-三次握手

  • SYN——SYN、ACK——ACK

传输层安全

  • 拒绝服务：syn flood/udp flood、smurf
  • 欺骗：TCP会话劫持
  • 窃听：嗅探
  • 伪造：数据包伪造

SYN洪水攻击

  • 原理：三次握手只进行到前两次，客户端syn，服务端syn、ack，但客户端迟迟不发ack，服务端就一直处于等待状态
  • 被攻击者主机一直处于等待状态，耗费大量资源，其再也收不到正常网络包，只有dos包
  • syn flood攻击包的源地址都是伪造的，追查工作难
  • 被攻击主机积累了大量的syn半连接，可用netstat查看：netstat -an | grep SYN
  • SYN洪水攻击比较难以防御，如下几种解决办法
    • 缩短SYN timeout时间（等待时间太长就不等了
    • 设置SYN cookie
    • 负反馈策略
    • 退让策略
    • 分布式DNS负载均衡
    • 防火墙

UDP flood

  • 大量UDP小包冲击应用服务器（DNS、radius认证等
  • 利用系统服务形成流量（echo chargen
  • 利用正常UDP服务发送大流量形成网络拥塞
  • UDP洪水主要是利用主机能自动进行回复的服务（例如使用udp协议的chargen服务和echo服务）来进行攻击
  • 很多提供www和mail等服务的，一般是unix服务器，其默认打开一些被黑客恶意利用的UDP服务；如echo服务会显示接收到的每一个数据包，而原本作为测试功能的chargen服务会在收到每一个数据包时随机反馈一些字符

LAND攻击

  • 运用IP Spoofing技术送出一连串SYN数据包给目标主机，让目标主机误以为这些数据包是由自己发送的
  • 由于目标主机在处理这些数据包的时候，他自己并无法回应给自己SYN-ACK数据包，因而造成系统宕机
  • 即构造一个数据包，目的地址和源地址均是被攻击者，其自己无法给自己回应，故……
  • Land是因特网上最常见的拒绝服务攻击类型，由rootshell发现
  • 原理：向目标即发送大量的源地址和目的地址相同的包，造成目标机解析Land包时占用大量的系统资源，从而使其瘫痪

1.5-应用层安全

应用层协议

  • 定义了，运行在不同端系统上的，应用程序，如何相互传递报文
  • 常见：域名解析-DNS、电子邮件-SMTP和POP3、文件传输-FTP、网页浏览-HTTP

安全问题-明文

  • 截获流量，得到明文传输的敏感信息

DNS欺骗

  • 正常DNS查询：递归、迭代
  • 欺骗：
    • 用户a查询test网站的IP
    • 向dns1查询
    • dns1不知道，继续向dns2查询
    • dns2知道，返回结果1.1.1.1
    • 攻击者修改为2.2.2.2
    • 故dns1得到为2.2，故用户a得到也为2.2
    • 多个用户向dns1查询，故其可以影响多个用户

安全问题

  • 拒绝服务：超长URL链接
  • 欺骗：跨站脚本、钓鱼式攻击、cookie欺骗
  • 窃听：数据泄露
  • 伪造：应用数据篡改
  • 暴力破解：应用认证口令暴力破解
