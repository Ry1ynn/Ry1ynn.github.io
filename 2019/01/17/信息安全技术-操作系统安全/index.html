信息安全技术-操作系统安全

发表于 2019-01-17 | 分类于 NISP二级

1.1-操作系统安全概述

操作系统的功能

  • 用户与计算机硬件之间的接口
  • 操作系统为用户提供了虚拟计算机，把硬件的复杂性与用户隔离
  • 计算机系统的资源管理者
    • CPU管理
    • 存储管理
    • 设备管理
    • 文件管理
    • 网络与通信管理
    • 用户接口

操作系统安全目标

  • 标识系统中的用户，进行身份鉴别
  • 依据系统安全策略对用户的操作进行访问控制，防止用户和外来入侵者对计算机资源的非法访问
  • 监督系统运行的安全性
  • 保证系统自身的安全和完整性

操作系统安全机制

  • 标识与鉴别

    • 用户身份合法性鉴别
    • 操作系统登录

  • 访问控制

    • 防止对资源的非法使用
    • 限制访问主体对访问客体的访问权限
    • DAC、MAC、RBAC

  • 最小特权管理

    • 限制、分割用户和进程对系统资源的访问权限
    • 必不可少的权限

  • 信道保护

    • 可信通路（trusted path
    • 安全键（SAK

  • 安全审计

    • 对系统中有关安全的活动进行记录、检查以及审核
    • 审计一般是一个独立的过程

  • 内存存取保护

    • 进程间/系统进程内存保护
    • 段式保护、页式保护和段页式保护

  • 文件系统保护

    • 分区
    • 文件共享

1.2-Windows系统安全机制

系统标识与鉴别-安全主体

  • 安全主体类型
    • 用户账号
      • 本地用户
      • 域用户
    • 组账号
      • everyone组
      • network组
    • 计算机
    • 服务
  • 安全标识符（security identify，SID
    • 是安全主体的代表，标识用户、组和计算机账户的唯一编码
    • 范例：S-1-5-21-213213……（whoami /user可查看

用户鉴别

  • 登录信息管理体制
  • 登录验证
    • 本地登录验证
    • 远程登录验证
  • 账号信息存储（SAM：安全账号管理
    • 运行期锁定、存储格式加密
    • 仅对system账号有权限，通过服务进行访问控制
  • Windows用户身份鉴别：本地登录
    • GINA，Graphical identification and authentication，图形化识别和验证
    • LSA，local security authority，本地安全授权
    • 过程：账号密码—GINA—LSA—SAM—本地信息库对比—登录成功
  • 远程登录鉴别协议
    • SMB，server message block，口令明文传输
    • LM，LAN manager，口令哈希传输，强度低
    • NTLM，NT LAN manager，提高口令散列加密强度、挑战/响应机制
    • Kerberos：为分布式网络提供单一身份验证

系统访问控制-ACL

  • 访问控制列表，access control list，ACL
    • NTFS文件系统支持
    • 权限存储流文件系统中
    • 自主访问控制
    • 灵活性高，安全性不高

系统访问控制-用户账号控制

  • 用户账号控制，user account control，uac
    • 完全访问令牌
    • 标准受限访问令牌

文件系统安全

  • NTFS文件系统权限控制（ACL
    • 文件、文件夹、注册表键值、打印机等对象
  • 安全加密
    • EFS，encrypting file system
      • Windows内置，与文件系统高度集成
      • 对Windows用户透明
      • 对称与非对称算法结合
    • bitlocker
      • 对整个操作系统卷加密
      • 解决设备物理丢失安全问题

系统审计机制

  • Windows日志
    • 系统
    • 应用程序
    • 安全
    • 设置
  • 应用程序和服务日志
  • 应用访问日志
    • FTP访问日志
    • IIS访问日志

系统安全策略

  • 账户策略
    • 密码策略
    • 账户锁定策略
  • 本地策略
    • 审核策略
    • 用户权限分配
    • 安全选项

系统安全配置

  1. 安全配置前置工作
    • 安全的安装
      • 分区设置：不要只使用一个分区
      • 系统补丁：SP+hotfix
    • 补丁更新设置
      • 检查更新
      • 自动下载安装或手动
  2. 账号安全设置
    • 保护账号安全
      • 默认管理账号administrator更名
      • 设置好的口令：自己容易记，别人不好猜
    • 系统账号策略设置
      • 密码策略：避免系统出现弱口令（如长度最小值、最短使用期限、强制密码历史，即使用几次的历史密码不允许使用
      • 账号锁定策略：应对口令暴力破解（如账号锁定时间、账号锁定阀值、重置账号锁定计数器
    • 账号安全选项设置
    • 账号权限控制-用户权限分配
      • 从网络访问这台计算机
      • 拒绝从网络访问这台计算机
      • 管理审核和安全日志
      • 从远程系统强制关机
    • 账号权限控制-设置唤醒密码
      • 设置唤醒计算机时的登录密码
      • 设置屏幕保护恢复时的登录密码
  3. 关闭自动播放
    • 自动播放功能：为方便用户而设计
    • 恶意代码借助存储介质传播的方式
    • 关闭自动播放：可以有效阻断U盘病毒的传播路径
  4. 远程访问控制
    • 网络访问控制
      • 防火墙设置：确保启用Windows自带防火墙
      • 防火墙高级配置
        • 出战规则
        • 入站规则
        • 连接安全规则
        • 监视
          • 防火墙
          • 连接安全规则
          • 安全关联
    • 共享安全防护
      • 共享相关风险
        • IPC$的安全问题（空会话连接导致信息泄漏
        • 管理共享风险（远程文件操作
        • 普通共享的风险（远程文件操作
      • 关闭管理共享
        • 空会话连接控制：本地安全策略设置中对匿名访问的限制
        • 关闭管理共享：修改注册表
  5. 本地安全策略
    • 审核策略
    • 用户权限分配
    • 安全选项（位置：管理工具-本地安全策略-安全设置-本地策略-安全选项
      • 交互式登录：无须按ctrl+alt+del，设置为已禁用
      • 交互式登录：不显示最后的用户名，设置为已启用
      • 设备：将CD-ROM的访问权限仅限于本地登录的用户，设置为已启用
      • 网络访问：不允许SAM账号的匿名枚举，设置为已启用
      • 网络访问：可匿名访问的共享，删除策略设置里的值
      • 网络访问：可匿名访问的命名管道，删除策略设置里的值
      • 网络访问：可远程访问的注册表路径，删除策略设置里的值
  6. 服务运行安全设置
    • Windows服务：win服务程序是一个长时间运行的可执行程序，不需要用户的交互、登录
    • 关闭不需要的：如计划任务、远程操作注册表
    • 控制服务权限：system（本地系统）、local service、network service
  7. 使用第三方安全增强软件
    • 防病毒软件
      • 要安装：恶意代码是终端安全的最大威胁
      • 确保病毒库更新：病毒防护软件主要工作机制是特征码扫描
      • 开启云查杀
    • 系统安全防护软件
      • 系统安全防护软件
      • 影子系统
      • 主机入侵检测

1.3-Linux系统安全机制

系统标识与鉴别

  • 安全主体
    • 用户：身份标识（User ID
    • 组：身份标识（Group ID
  • 用户与组基本概念
    • 文件必须有所有者
    • 用户必须属于某个或多个组
    • 用户与组的关系灵活（一对多、多对多都可以
    • root用户拥有所有权限
  • 账号信息存储
    • 用户信息
      • /etc/passwd（账号
      • /etc/shadow（密码
    • 组信息
      • /etc/group
      • /etc/gshadow
  • 用户信息文件-passwd
    • 用于存放用户信息
    • 特点：文本格式、全局可读
    • 存储条目格式：name:coded-passwd:UID:GID:userinfo:homedirectory:shell
  • 用户账号影子文件-shadow
    • 用于存放用户密码散列、密码管理信息等
    • 特点：文本格式、仅对root可读可写
    • 存储条目格式：name:passwd:lastchg:min:max:warn:inactive:expire:flag
  • 口令鉴别
    • 本地登录
    • 远程登录（telnet、FTP等
  • 主机信任（基于证书
  • PAM（可插拔验证模块

系统访问控制-权限模式

  • 文件/目录权限基本概念
    • 权限类型：读、写、执行
    • 权限表示方式：模式位
    • 如：drwxr-xr-x
  • 权限的数字表示法：rwx=111=7，r-x=101=5
  • 权限的特殊属性：SUID（临时获得文件属主权限，故用户可以更改自己密码，而原本用户对改密码文件是没有权限的）、SGID、Sticky（防删除

安全审计-日志系统

  • 系统日志类型
    • 连接时间日志
      • /var/log/wtmp和/var/run/utmp
      • 由多个程序执行，记录用户登录时间
    • 进程统计
      • 由系统内核执行，为系统基本服务提供命令使用统计
    • 错误日志
      • /var/log/messages
      • 由syslogd守护记录，制定注意的事项
  • 应用程序日志：如HTTP、FTP等创建的日志

文件系统

  • 文件系统类型
    • 日志文件系统：Rxt4、Ext3、XFS、JFS等
  • 文件系统安全
    • 访问权限
    • 文件系统加密
      • eCryptfs（Enterprise Cryptographic Filesystem
      • 基于内核，安全性高，用户操作便利
      • 加密元数据写在每个加密文件的头部，方便迁移、备份

特权管理

  • 特权划分
    • 分割管理权限，30多种管理特权
    • 根用户（root）拥有所有特权
    • 普通用户特权操作实现：setuid、setgid
  • 特权保护：保护root账号
    • 不直接使用root登录，普通用户su成为root
    • 控制root权限使用

系统安全配置

  1. 安全配置前置工作
    • 系统安装
      • 使用官方/正版软件
      • 分区挂载重要目录：根目录/，用户目录/home，临时目录/tmp等应该分开到不同的磁盘分区
      • 自定义安装，选择需要的软件包：不安装全部软件包，尤其是那些不需要的网路服务包
    • 系统补丁
      • 及时安装系统补丁
      • 更新补丁前，先在测试系统上对补丁的可用性和兼容性进行验证
  2. 账号和口令安全
    • 账号通用配置
      • 检查、清除系统中多余账号（检查：cat /etc/passwd、cat /etc/shadow
      • 锁定账号
        • 特殊保留的系统伪账户，可以设置锁定登录
        • 锁定命令：#passwd -l 用户名
        • 解锁命令：#passwd -u 用户名
      • 禁用root之外的所有超级用户
        • 打开系统账号文件/etc/passwd
        • 若用户ID=0，则表示该用户拥有超级用户的权限
        • 检查是否有多个ID=0
        • 禁用或删除多余的账号
      • 检查是否存在空口令账户
        • 执行命令：awk -F:’(== “”){print $1}’ /etc/shadow
        • 如果存在空口令账户，则对其进行锁定，或要求增加密码
        • 要确认空口令账户是否和已有应用关联，增加密码是否会引起应用无法连接的问题
      • 设置账户锁定登录失败锁定次数、锁定时间（system-auth文件
      • 修改账户超时值，设置自动注销时间（profile文件
    • 保护root账号
      • root账号权限很高
      • 保护措施
        • 禁止使用root登录系统
        • 只允许普通用户登陆，可通过su命令切换到root
        • 不随意把root shell留在终端上
        • 不把当前目录./和普通用户的bin目录放在root账号的环境变量PATH中
        • 不以root运行其他用户的或不熟悉的程序
    • 口令安全策略
      • 使用安全口令：口令长度、字符要求
      • 口令修改策略
        • 强制口令使用有效期
        • 设置口令修改提醒
        • 设置账户锁定登录失败锁定次数、锁定时间
  3. 系统服务配置
    • 禁止危险的网络服务：telnet、FTP、echo、chargen、shell等
    • 关闭非必需的网络服务：talk、ntalk等
    • 确保最新版本：使用最新的、最安全的版本
  4. 远程登录安全
    • 禁用telnet，使用SSH进行管理
    • 限制能够登录本机的IP地址（sshd_config文件
    • 禁止root用户远程登录（sshd_config
    • 限定信任主机
    • 修改banner 信息
  5. 文件和目录安全
    • 保护重要的文件目录，限制用户访问
      • 设置文件的属主和属性以进行保护
      • 重要的文件或目录设置为不可改变属性：chattr +i /etc/passwd
    • 临时文件不应该有执行权限
    • 设置新创建文件的默认权限掩码：可根据要求设置新文件的默认访问权限，如仅允许文件属主访问，不允许其他人访问
    • umask设置的是权限补码
    • 设置方法：
      • 使用umask命令，如#umask 066
      • 编辑/etc/profile文件，设置umask值
    • 检查SUID/SGID文件
      • SUID/SGID的程序在运行时，将有效用户ID改变为该程序的所有者（组）ID，因为可能存在一定的安全隐患
      • 找出系统中所有含s“位的程序，把不必要的s位去掉，或者把根本不用的直接删除，可防止用户滥用及提升权限的可能性
      • 查找SUID可执行程序#find / -perm -4000 -user 0 -ls
      • 查找SGID程序 # find / -perm -2000 -user 0 -ls
  6. 系统日志配置
    • 审查日志中不正常情况
      • 非常规时间登录
      • 日志残缺
      • Su的使用
      • 服务的启动情况
    • 启用syslog服务
    • 配置日志存储策略：打开syslog文件，检查其对日志存储空间的大小和时间的设置
    • 使用syslog设备：syslog.conf设置、使用syslog日志服务器
  7. 使用安全软件
    • 启用主机防火墙
    • 使用最新版本安全软件：用官方的、最新的
    • Linux下的防火墙框架iptables
      • 已内嵌到Linux系统中、功能较强大
      • 功能：包过滤、NAT、数据包处理
      • 建议设置开机自启
