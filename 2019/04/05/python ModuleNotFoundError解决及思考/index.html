python ModuleNotFoundError解决及思考

发表于 2019-04-05 | 分类于 乱七八糟

01-相关知识点

  • 当你导入一个模块，Python解析器对模块位置的搜索顺序是：
    • 当前目录
    • 如果不在当前目录，Python则搜索在shell变量PYTHONPATH下的每个目录。
    • 如果都找不到，Python会察看默认路径。UNIX下，默认路径一般为/usr/local/lib/python/
    • 模块搜索路径存储在system模块的sys.path变量中。变量里包含当前目录，PYTHONPATH和由安装过程决定的默认目录。
  • 查看当前shell：echo $SHELL
  • 更改shell：cash -s /bin/zsh
  • 不同shell有不同的配置文件
    • bash对应~/.bash_profile
    • zsh对应~/.zshrc
  • PYTHONPATH在shell的配置文件中定义，如export PYTHONPATH="xxx/yyy"
  • .zshrc修改后要重启系统后生效，若要立即生效source ~/.zshrc
  • pip3 show pandas，查看其详细信息，可得其所在目录

02-实操

  1. pip3 show pandas得Location: /usr/local/lib/python3.7/site-packages

  2. echo $SHELL得当前shell为/bin/zsh

  3. vim ~/.zshrc（原先并不存在，可由bash_profile复制而来）

  4. source ~/.zshrc使配置生效

  5. 经测试，问题解决。
