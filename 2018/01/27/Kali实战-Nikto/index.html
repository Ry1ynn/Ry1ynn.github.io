Kali实战-Nikto

发表于 2018-01-27 | 分类于 Kali相关

扫描工具之Nikto

简介

  • perl语言开发，开源web扫描器
  • 可以扫web服务器的软件版本，如apache，tomcat
  • web server也是其关注点，不仅仅针对代码层
  • 搜索存在安全隐患的文件，如备份文件
    • 渗透是黑盒测试
    • 若发现存在的源代码，白盒
  • 扫描服务器的配置漏洞
    • 默认配置不一定是安全配置
    • 默认配置问题已列入top10，较严重
  • web application层面的安全隐患，代码层面，如sql注入，xss

404误判问题

  • web扫描器的根本任务是找到所有的页面，理想情况200和404来判断
  • 很多服务器并不遵守RFC标准，对于不存在的对象也会返回200，因此扫描不准确
  • 为解决上述问题，可以根据文件内容来判断，但是不同后缀的文件的404返回内容是不同的
  • 为解决新问题，扫描目标前，会进行不同后缀名的尝试，得到其对应的返回内容
    • 所有扫描前的准备工作
    • 后缀名存在于nikto的数据库中
    • 随机名字加后缀拼接成文件名，去请求
    • 这个文件是随机的，一定不会存在于目标
    • 返回不存在时的信息
    • 将信息中那些时间等动态信息去除，进行md5，得到不存在时的返回信息，视为一个标准
    • 后面扫描时，将信息与不存在时返回的信息进行对比，从而准确判断
  • -no404参数
    • 默认是不加的
    • 避免前期尝试的准备工作，直接扫描
    • 可以提高扫描效率
    • 相应的会降低准确率
    • 误判可能性大，不建议

常用命令

  • -update
    • 升级数据库及插件
  • -list-plugins
    • 列出插件，笼统的大类
  • -useproxy
    • 使用代理
  • -host
    • 指定
    • 可以是IP，也可以IP：port
    • 可以域名，并指定目录
    • 可跟文件，内含多个目标
  • -port
    • 指定端口
    • 默认扫描80
    • 可跟多个端口
  • -ssl
    • ssl的模式
  • 结合nmap扫描
    • nmap扫描结果作为其目标
    • 利用Linux管道命令
    • nmap -p 80 192.168.1.1/24 -oG - | nikto -host -，-表示前者的结果
  • -vhost
    • 指定虚拟主机
    • 一个IP存在多个网站
    • 只能跟域名，不可IP

扫描时与用户交互

  • 空格，显示当前扫描状态
  • v，verbose，显示当前正在扫描的信息，再按关闭，下同
  • d，debug，更详细的信息
  • e，error，错误信息
  • p，progress，进程信息
  • r，redict，显示重定向
  • 处，cookie显示cookie
  • ａ，auth，认证信息
  • q，推出扫描
  • N，下一个host，针对多个目标时
  • Ｐ，暂停

扫描时认证问题

  • 很对扫描会有认证过程，成功后才可以进一步，如登录表单
  • 两种认证方式
    • 基于webserver的，不常见，多见于嵌入式设备
    • 基于web application，代码层面，如登录页面
  • -id参数可解决基于webserver时的http认证，不适合代码层面
  • 对于登录表单
    • nikto本身不支持表单认证
    • 可采取cookie方式，效果一样
      • 不提供相关参数来指定cookie
      • 利用配置文件
      • 浏览器登录表单，获取cookie
      • 将cookie写入其配置文件
      • 如此，就会携带cookie扫描，从而通过认证

配置文件

  • /etc/nikto.conf
  • 修改user-agent
    • 躲避报警机制
    • 避免目标日志记录实施攻击的具体工具
  • 修改cookie
    • 实现代码层的认证
    • 每次扫描请求都会带着cookie去访问
  • 修改RFIURL
    • 测试远程文件包含
    • 可修改为自己服务器上的
  • CIRT
    • 更新时从这个IP获取信息
    • 官方IP是需要翻墙的
  • 代理设置
    • 修改后，则不需要使用-useproxy
    • 一个是临时，一个是永久

躲避IDS技术

  • -evasion参数
  • 共有8种方法，man nikto可见
  • 可同时使用多个
  • 以数字ID表示采取的方法
  • 如-evasion 145
