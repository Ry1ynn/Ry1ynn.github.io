Kali实战-Skipfish

发表于 2018-01-27 | 分类于 Kali相关

扫描工具之Skipfish

简介

  • 命令行工具
    • c语言编写
    • 无代理功能，只可主动扫描
  • 实验性的主动web安全评估工具
    • 实验性，非商业版
    • Google的实验性项目不必其他商业软件差
  • web application的代码层面
  • 两种大方法
    • 给出，起始页面，递归爬网
    • 基于字典的探测，做fuzz
  • 优势
    • 相对而言，误报较低
    • 速度较快
      • 多路单线程，全异步网络i/o，消除内存管理和调度开销
      • 启发式自动内容识别

常见使用

  • -o test 目标URL
    • o参数指定结果输出的目录，必须的参数
  • -o test @url.txt
    • 指定多个目标
  • -o test -S complete.wl -W a.wl 目标URL
    • 一般是根据页面内容URL来递归爬取
    • 可能会有隐藏的，故S参数指定字典，基于字典的扫描
    • W参数，扫描出目录及其他特征字符，指定文件来存放，方便后续使用
  • I参数，只检查包含某字符串的URL
    • 规定扫描范围
  • X参数，不检查包含某字符串的URL
    • 如logout
  • K不对指定的参数进行fuzz测试
  • D参数，跨站点爬另一个域
    • 扫描过程中通过网页内容的超链接可能会得到其他域名的网页，会超出测试范围，不应扫描
    • 有时，也需要去扫描
      • 如，网页存在嵌套，a网页中有b域名的网页
      • 前提是扫描时通过网页内容的超链接得到了其他域名的网站，没有得到，出于测试范围，也不会去扫描
    • K参数指定了需要扫描的其他域名，指定了且爬取得到的才可以扫描，并且关系，缺一不可
  • 速度相关
    • l，每秒最大请求数
    • m，每个IP最大并发连接数
    • 避免触发报警机制
  • —config，指定配置文件
    • 不用每次加这么多参数
    • 一次修改，多次使用

身份认证相关

  • 目标服务器需要http基本身份认证，web server层面
    • A参数，如-A user:pass
  • 扫描时遇到登录页面，web application 代码层面
    • cookie方法
      • C参数，如-C “k1=v1” -C “k2=v2”
    • 表单登录方法
      • —auth-form
        • 表单所在的URL
      • —auth-user
        • 提交的用户名
      • —auth-pass
        • 提交的密码
      • —auth-verify-url
        • 登录成功后跳转的URL，用于验证登录是否成功
      • 其他三个相关参数，使登录更准确，提高登录成功率
        • —auth-form-target
          • form表单的action内容
          • 注意是绝对路径
        • —auth-user-field
          • form表单中用户名字段
        • —auth-pass-field
          • 密码字段名称
