Kali实战-nmap

发表于 2018-01-27 | 分类于 Kali相关

主动信息收集_nmap

target specification

  • -iL
    • -iL ip.txt
    • 从文件中获取待扫描的IP
  • -iR
    • -iR 100:随机选取100个IP
    • 随机选取，不用非得自己网段，世界各地的
  • —exclude/excludefile
    • —exclude 1.1.1.1-30
    • 排除掉指定的的IP（或IP段
    • excludefile，即排除文件中的IP

host discovery

  • -sL
    • 不做扫描，仅把待扫描的IP列出来（或列出ip.txt中目标，意义不大
    • -sL x.x.x.0/28，即列出16个地址
    • 功能类似子网掩码计算IP个数
  • -sn
    • 不做端口扫描
    • 用于主机发现层面
  • -Pn
    • 跳过主机发现阶段（认为所有主机都存活
    • 扫描防火墙时常用到
    • 目标中存在防火墙等特殊设备，若不加-Pn，在主机发现时，其没有回包，nmap以为其主机不存活，故不进行下一步的端口扫描，但其上可能存在某开放端口，因此会错过这一存活主机
  • -PS/PA/PU/PY
    • 利用tcp syn/ack和udp和sctp
    • sctp不是tcp/ip范畴内的协议
  • -PE/PP/PM
    • ICMP echo，timestap，netmask
    • netmask查对方的子网掩码，很少查到，除非安全性很烂
  • -P0
    • IP协议ping的扫描
  • -n/-R
    • -n：禁止做dns解析（包括域名到IP的正向解析和IP到已经的逆向
    • -R：总对目标IP实施dns的逆向解析
  • —dns-servers
    • 在做dns解析时，自定义dns服务器
    • 如新浪有智能dns功能，中国访问与美国访问，得到的IP是不同的
    • 自定义dns即可模拟不同地区访问目标
  • —system-dns
    • 使用系统自带的dns，加不加无所谓
  • —traceroute
    • 扫描时进行路由追踪
    • 类似traceroute命令

scan technique（针对端口扫描的技术

  • -sS/sT/sA/sW/sM
    • 默认是tcp的syn扫描
    • sT是tcp connect，建立完整连接的扫描，结果较准确
    • sA便是tcp的ack扫描
    • sW是tcp中window，即窗口扫描
    • sM是maimon，也是tcp的范畴
    • 皆是是tcp不同flag的组合，（如-sA为syn+ack，不确定）Wireshark抓包，分析其tcp结构中不同flag的组合，对应的哪个参数
  • -sU
    • udp扫描
  • -sN/sF/sX
    • tcp null ，FIN，Xmas
    • sN即无flag，flag位全为0
  • —scanflags
    • 前面的很多参数，都是tcp的不同flag组合，nmap自带的
    • 若不满已有的，可自定义flag组合
    • 也不是随意组合的，要很懂网络，自己搭配的flag能够判断目标状态，否则无意义
  • -sI
    • 僵尸机扫描
  • -sY/sZ
    • 针对SCTP的，SCTP INIT/COOKIE-ECHO
    • 用的不多，不常见
  • -sO
    • IP协议的扫描
  • -b
    • ftp bounce 扫描，ftp中继的场景

port specification and scan order

  • 指定要扫什么端口
  • -p
    • 指定扫描的端口
    • 可以单个端口，可以-表连续端口，可以,表不连续端口
    • 可以指定是tcp端口还是udp端口，若不指定，则默认两种全选
    • 举例
      • -p53
      • -p1-65535
      • -p U:53,111,137, T:21-25,80
  • —exclude-ports
    • 排除指定端口
  • -F
    • 快速扫描模式
    • 如不加p指定端口，则会扫描默认的1000个常用端口，若加F，则会扫描较少的端口
  • -r
    • 连续扫描端口，按数字顺序
    • 默认情况下是随机的，不按数字顺序的
  • —top-ports
    • 不加任何端口扫描相关的参数时，默认扫描目标机器的1000个常用端口
    • 若—top-ports 10，则只扫1000中前10个
  • —port-ratio
    • 用的不多，自行查阅

service/version detection

  • -sV
    • 目标端口跑的服务或其版本
    • 与特征库中的指纹相对比，得到结果
  • —version-intensity
    • 默认情况，与特征库比对，但不会全部比对，是比较粗略的
    • 若想精确探测其版本，则可用此参数，探测强度0-9
    • 需要与-sV相结合使用
  • —version-light
    • 即探测强度为2的情况
  • —version-all
    • 即探测强度为9的情况
    • 最强的是9
  • —version-trace
    • 追踪其探测过程，将其详细信息打印出来

script scan

  • -sC
    • 使用默认的脚本
    • 等于—script=default
  • —script
    • 指定使用的脚本
  • —script-args
    • 给所使用的脚本提供参数
  • —script-args-file
    • 提供的脚本参数在文件里
  • —script-trace
    • 追踪过程，即将详细信息打印出来
  • —script-updatedb
    • 更新脚本数据库
  • —script-help
    • 指定某一脚本，显示其帮助信息

OS detection

  • -O
    • 启用系统探测
  • —osscan-limit
    • 限制nmap只对确定的主机进行os探测
    • 至少需确知该主机分别有一个open和closed的端口
  • —osscan-guess
    • 大胆猜测对方主机的系统类型
    • 由此准确性会下降

timing and performance

  • -T
    • 指定扫描的速度
    • 0-5，越大越快
  • —min-hostgroup/max-hostgroup
    • 并行扫描时，控制每一组的最大主机数目（或最小
    • 很少使用，但都用于保持组的大小在一个范围内
  • —min-parallelism/max-parallelism
    • 调整探测报文的并行度，即并行的数量
  • —min-rtt-timeout/max～/initial～
    • 设置报文超时的时间（等待探测报文响应的时间
    • 若超过，则放弃
  • —max-retries
    • 实际情况，可能会因为网络状况导致包丢失，则会retry
    • 设置最大重新探测的次数
  • —host-timeout
    • 设置等待主机响应德时间
    • 放弃低速目标主机
  • —scan-delay/—max-scan-delay
    • 发送每个探测请求的间隔时间
    • 频繁探测会使目标觉察
  • —min-rate/max-rate
    • 每秒发送包的最小最大数量

firewall/ids evasion and spoofing

  • -f；—mtu
    • -f指定报文分段数
    • —mtu指定mtu，即最小传输单元
  • -D
    • 混杂进一些虚假IP，增加噪声，使目标难以分辨真正的nmap主机
    • nmap -D ip1,ip2… 目标IP
  • -S
    • 欺骗源地址
    • 若于目标主机位于同一网段，则其可知自己Mac地址，故可收到响应（外网呢？
    • 存在明显弊端，（具体bd
    • 一般与-e结合，指定发送的接口，如-e eth0
  • -e
    • 指定出口网卡
  • -g/–source-port
    • 指定扫描时的源端口，-g10000
  • —proxies
    • 指定代理服务器
  • —data
    • 在请求包的数据位置，加入自定义数据
    • 十六进制
  • —data-string
    • 加自定义的ascii string
  • —data-length
    • 发送报时，附加指定数量的随机字节数据
  • —ip-options
    • 制定自定义的IP选项
  • —ttl
    • 发送IP ttl field
  • —spoof-mac
    • 欺骗mac地址，让目标误以为是别的mac发出的
  • —badsum
    • 故意发送错误的chksum包（chksum用于校验）

misc

  • -6
    • 启用ipv6
  • -A
    • 启用系统探测、版本探测、脚本扫描及traceroute等
