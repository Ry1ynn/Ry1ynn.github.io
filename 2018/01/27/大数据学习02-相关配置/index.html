2.5-配置ssh免密码登录

  • 配置ssh免密码登录是让各节点能自动通过安全认证，不影响正常通讯

  • 在线安装，apt-get install ssh

  • 查看ssh运行状态，serrvice ssh status

  • 无密码登录思路

    1. 在每台机子上都使用ssh-keygen生成public key，private key
    2. 所有机子的public key都拷到一台机子如master上
    3. 在master上生成一个授权key文件authorized_keys
    4. 最后把authorized_keys拷给所有集群中的机子，就能保证无密码登录

  • 简要步骤

    1. 先在master上，在当前用户目录下生成公钥、私钥对

    2. 将公钥文件内容导入至authorized_keys文件

      cat .ssh/id_rsa.pub >> .ssh/authorized_keys

    3. 测试连接本机是否成功，ssh master，若不需要密码，则成功

      • 若仍需要密码，则检查authorized_keys文件权限，并做相应修改
      • exit退出当前的ssh会话
    • 常见问题及解决
      • 请先检查SSH服务是否启动，如果没启动，请启动
      • 如果没有.ssh目录则创建一个
      • 如无权限，使用命令修改要操作文件夹的owner为当前用户

    1. 其它机器上生成公钥、密钥，并将公钥文件复制到master（以slave_1为例

    2. 此时，master上便得到了其他两主机的公钥文件，分别将其导入至authorized_keys

      这样，master这台机器上，就有所有3台机器的公钥了

    3. 将master上的“最全”公钥，复制到其它机器

    4. 验证：每个虚拟机分别测试ssh登录其它，看是否需要密码ssh master/slave1/slave2

  • 步骤小结：

    • 在各节点上生成公钥、私钥
    • 将公钥发放其它所有节点
