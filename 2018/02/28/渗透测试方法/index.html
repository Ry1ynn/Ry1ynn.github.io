kali Linux渗透测试方法

安全问题的根源

  • 从更加宏观的角度看待安全
    • 概览安全体系的知识结构
    • 明确渗透测试在其中的定位
  • 最近一次技术交流引起的思考
  • 分层思想的利弊
    • 片面的认识安全
    • 只追求功能的实现
    • 人这一层面是问题的根源
    • 静态死板的分层并不能满足所有安全的需求

软件安全生命周期

  • 需求
    • 安全需求
  • 设计
    • 系统架构
  • 编码
    • 编码规范
  • 测试
    • 代码审计
  • 上线
    • 部署规范
  • 运维
    • 环境审计

渗透测试的意义

  • 安全建设
    • 周期长
    • 投入大
    • 效果不尽人意
  • 渗透测试
    • 相对软件安全生命周期，逆向思维
    • 从问题出发检查系统安全
    • 用黑客角度审视系统
    • 在资源有限的条件下效率更高
      • 中小型企业
      • IT资金投入不足，安全更不足
    • 渗透测试的目标是让系统更安全
      • 而非恶意黑客
      • 部分白帽子嘲笑某系统安全方面辣鸡现象

Kali Linux简介

  • 前身是BackTrack
  • 包含600+工具，仍需自己添加
  • 安全从业人员专用操作系统
  • 安全工具军火库
  • 支持多平台，包括ARM
  • 渗透测试和安全审计的平台
  • 工具党问题
    • 只掌握工具不掌握原理
    • 编程语言，扫描工具其实都是工具
    • 不可能从0101代码层面考虑，编程语言本身也是工具
    • 掌握工具使用仍是重要的

渗透测试标准PETS

  • 前期沟通
    • 不涉及技术层面
    • 渗透测试的时间，地点
    • 使用的工具，ddos抗压测试工具
    • 目标范围，通常给出域名，是否包括其子域名
  • 信息收集
    • 技术层面
    • 被动信息搜集（OSINT）
      • 域名，人员，邮箱，地址，手机号码等
        • 为了定制专属的字典，密码破解
        • 本地工具
          • nslookup、dig、whois
          • Fierce
            • 较综合工具
              • Dns查询
              • 区域传输
              • 基于字典的解析记录暴力破解
              • Subtopic
          • Dnsrecon
            • 功能类似上者
        • 在线网站
          • 站长之家等
      • 搜索引擎
        • shodan
          • 针对联网设备
          • 例如，查看目标是否使用某防火墙
          • 搜索联网摄像头
        • 谷歌黑客
      • Metadata
        • 拍摄设备拍摄的照片会泄露信息，如GPS
      • 专属密码字典
        • 工具Cupp，自行安装，基于py
        • 基于之前搜集的信息
    • 主动信息搜集
      • 扫描IP地址段
        • 工具nmap
        • 思路
          • 域名
          • 其子域名
          • 其IP地址
          • 其c段IP地址
          • IP地址对应主机的端口开放情况
          • 端口对应服务的版本
          • 根据版本搜索漏洞库寻找已知漏洞
          • 根据漏洞寻相应payload
        • 上步骤可写shell脚本，所有命令集中在同一脚本，自动化
      • 其他服务扫描
        • 除nmap扫描的中一般服务外的其他服务
        • SNMP、SMB、SMPT、WEB
          • Snmp
            • Onesistyone命令
            • Snmpwalk工具
      • 识别防护机制
        • LBD工具
          • 测试负载均衡
        • Wafw00f工具
          • 测试waf是否存在
        • Fragroute工具
          • 是否有其他IDS，IPS
        • Nmap
  • 威胁建模
    • 传统威胁建模方法
      • 资产/攻击者
    • PETS标准威胁建模方法
      • 按照PETS，一般是非黑盒方面，不涉及技术，走访，问卷调查企业，获取相关信息
      • 商业资产
        • 主要资产
        • 次要资产
      • 商业流程
        • 技术支持
        • 资产管理
        • 人力资源
        • 第三方
      • 威胁主体
        • 内部
        • 外部
      • 威胁能力
        • 威胁主体的能力
        • 威胁的可能性
      • 企业专有的威胁模型，而非通常的技术模型
    • 动机建模
      • 企业内部人员威胁，其动机
    • 影响建模
    • 从技术黑盒的角度进行建模
      • 工具，技术，能力发现被攻击面
      • 确定实施渗透的最佳路径
      • 确定控制，流程，架构
    • 建立的模型结果作为渗透测试报告的一部分进行提交
  • 漏洞分析
    • 漏洞扫描
      • 系统层面
        • Nessus
        • Openvas
        • Nexpose
        • Nmap script
      • Web层面
        • Nikto
        • Burpsuite
        • Owasp ZAP
        • Sqlmap
    • 已知漏洞利用
      • 在线搜索
        • 在线搜索exp平台，如exploitdb
        • 搜索引擎
      • 本地工具
        • Searchsploit
        • Sandi
        • Armitage
        • Msf漏洞利用平台
  • 渗透攻击
    • 明显技术层面
    • 进行到此步发现有所不足，可能会重新建模，重新漏洞分析，反复进行，周而复始。
    • 未知漏洞挖掘
      • Fuzzing
      • Edb
      • Ollydbg
    • 常见遇到防护措施，如waf，如杀毒软件。 需要进行面杀编码绕过
      • Webshell编码和过滤绕过
      • Metasploit工具，msfvenom一般生成的payload会被杀软识别，虚编码绕过
      • Vile-evation，功能类似上，会对payload进行编码混淆。 Kali未自带。
    • Wifi渗透
      • 到目标办公地点进行
      • Wifite工具
    • 密码破解
      • Hydra
      • John
      • PTH
      • Subtopic
    • 社工
      • 侧面攻击
      • Setoolkit工具
  • 后渗透测试
    • 取得一台主机控制权后
    • 扩大战果内网渗透
      • 内网扫描
      • 抓包分析
      • 地址欺骗
    • 本地提权
    • 获取数据
    • 擦除痕迹
    • 留后门
  • 渗透测试报告
    • 管理层报告
      • 风险级别
      • 风险分类
      • 整改规划
    • 技术报告
      • 漏洞细节
      • 漏洞复现
      • 修补方案

Kali Linux渗透测试实践方法

  • 简单总结
    • 安全工作的目标是实现安全
    • 渗透测试的目的不是证明系统有多烂和测试者有多牛
    • PETS是安全实操的方法论，掌握标准但但不要迷信标准
    • 保持学习状态，重视理论研究，多做上手实践
