1.1-信息与信息安全

信息与信息技术

信息

  • 简介
    • 信息奠基人-香农认为：信息是用来消除随机不确定性的东西
    • 信息是事物运动状态或存在方式的不确定性的描述
    • 信息是具体的，并且可以被人（生物、机器等）所感知、提取、识别，可以被传递、储存等
    • 信息来源物质，而非物质本身；其从物质的运行产生，可脱离源物质而寄生于媒体物质，相对独立的存在
  • 信息的定义
    • 信息是有意义的数据，具有一定价值，是一种需要适当保护的资产
    • 数据是反映客观事物属性的记录，是信息的具体表现形式
    • 数据经过加工处理后成为信息
    • 信息需经过数字化处理转变为数据后，才可存储和传输
  • 信息的功能
    • 反映事物内部属性、状态、结构、相互联系以及与外部的互动关系，减少事物的不确定性
  • 信息的表达
    • 信息本身是无形的，借助信息媒体，以多种形式存在或传播
    • 可存储在计算机、磁带等介质中，也可记忆在人脑，还可通过网络等方式传播
  • 信息与消息
    • 信息不同于消息
    • 消息是信息外壳，信息是消息内核
    • 即，消息是信息的笼统概念，信息是消息的精确概念
  • 信息与数据
    • 信息不同于数据
    • 数据是信息的符号表示（或称载体），信息是数据的内涵、语义解释
    • 数据是信息存在的一种形式，通过解释或处理后才可成为有用信息
    • 数据可用不同形式表示，信息不会随数据不同的形式而改变

信息技术

  • 简介
    • information technology（IT），是管理、处理信息时所采用的各种技术的总称
    • 主要是利用计算机科学和通信技术，来设计、开发、安装和实施信息系统及应用软件
    • 信息处理：获取信息并对其进行变换，使之成为有用信息并发布出去的过程；包括信息获取、存储、加工、发布和表示等环节
    • 信息收集和处理是人类社会工作的重要组成部分
  • 信息技术两方面：生产和应用
    • 信息技术生产：体现在信息技术产业本身，包括计算机软硬件、设备制造，微电子电路等
    • 信息技术应用：体现在信息技术的扩散，包括信息服务、信息管理系统等

信息技术发展阶段

  • 微电子、通信、计算机和网络是信息系统核心，其发展进程四阶段：
    1. 电讯技术的发明
    2. 计算机技术的发展
    3. 互联网的使用
    4. 网络社会
  • 阶段2：电讯技术的发明
    • 信息技术起源可追溯到19世纪30年代，标志性事件：电报电话的出现
    • 电讯技术新时代：
      • 1835，美国人，莫尔斯Morse，发明电报
      • 1837，莫尔斯电磁式有线电报问世
      • 1886，马可尼，发明无线电报机
      • 1876，贝尔Bell，发明电话机
      • 1906，美国人，费森登，研究出无线电广播
      • 1912，美国，Emerson公司，制造出世界上第一台收音机
      • 1925，约翰·贝德，发明世界上第一台电视机
  • 阶段2：计算机技术的发展
    • 20世纪30年代，计算机理论与技术发展，信息技术进入计算机阶段
    • 50年代，出现第一代电子管计算机，用于军事科研信息处理
    • 60年代中，第二代晶体管计算机，在民用企业中使用
    • 60年代末，集成电路（integrated circuit，IC）、大规模集成电路计算机，在社会普及应用
  • 阶段3：互联网的使用
    • 20世纪60年代末，美国，第一个计算机网络ARPANET，用于军事目的；
      它使连接到网络上的计算机可以相互交流信息
    • 90年代，计算机网络发展为全球性网络——因特网（Internet），网络技术及应用迅猛发展；
      电话、计算机等设备实现了互联互通
      信息技术在此阶段迅速发展
  • 阶段4：网络社会
    • 20世纪末，Internet为核心的信息技术进一步发展，人们工作生活学习、国家经济、社会治理与网络密不可分
    • 一方面通过现实社会投射，构成虚拟的网络社会
    • 另一方面通过网络信息渗透，融合各种已存在的社会网络实体，使网络社会成为整个现实社会的结构形态，至此，信息技术进入崭新阶段——网络社会阶段
    • 此阶段，云计算、物联网、大数据等技术出现

信息安全

信息安全介绍

  • 简介
    • 信息安全指信息系统（软硬件、数据、人、物理环境及基础设施）受到保护，不受偶然或恶意原因的影响，而遭到破坏、更改、泄露
    • 系统连续可靠正常地运行，信息服务不中断，最终实现业务连续性
    • 信安根本目的：信息不受内部、外部、自然等因素的威胁
    • 所谓信息安全关注信息本身安全，而不管是否应用计算机作为信息处理手段
    • 我国信息安全标准化委员会：信息安全是指保持和维持信息的保密性、完整性和可用性，也有真实性、可核查性、抗抵赖性和可靠性
  • 信息安全的目标
    • 保证信息上述安全属性得到保持，不被破坏，对组织业务运行能力提供支撑
  • 信息安全的任务
    • 保护信息资产（信息和信息系统）免受未经授权的访问使用、披露、破坏等
  • 信息安全的属性
    • 信息安全
      • 基本属性
        • 完整性
        • 机密性
        • 可用性
      • 其他属性
        • 真实性
        • 可控性
        • 不可否认性

信息安全的属性

  • 从消息层次看，其属性有
    • 机密性：即保密性，控制信息资源的开放范围，确保信息在各过程中不会泄露给非授权用户、实体和进程，或被 其利用
    • 完整性：确保信息在各过程中保持未经授权不可改变的特性，即对抗主动攻击，保持数据一致，防止数据被非法用户修改或破坏
    • 不可否认性：即抗抵赖性，建立有效的责任机制，防止用户否认其发送信息的行为和信息的内容
  • 从网络层次看，其属性有
    • 可用性：确保授权用户、实体和进程对信息及资源的正常使用，而不会被不合理拒绝，允许其可靠而及时的访问信息及资源
    • 可控性：对信息传播范围及内容有控制能力，防止非法利用信息和系统

信息安全的特征

特征-1

  • 相对性
    • 安全是相对的，动态的，没有绝对安全的系统
    • 故需要及时对系统安全问题进行跟踪处理，定期进行整体安全评估，及时发现问题并加以解决
  • 时效性
    • 安全不可一劳永逸
    • 根据安全风险，及时制定应对策略
  • 相关性
    • 更改配置等操作时，对系统重新进行评估，同步更新安全措施

特征-2

  • 不确定性

    • 攻击发起的时间、地点、攻击者和攻击目标都具有不确定性，制定安全应对措施时，考虑所有潜在的安全威胁
  • 复杂性
    • 信息安全是一项系统工程，涉及安全管理、教育等多领域

4个鲜明特征（与传统安全相比）

  • 系统性

    • 系统地从各层面综合保障信息安全

  • 动态性

    • 不能一劳永逸
    • 在信息系统整个生命周期，采取相应措施，控制风险

  • 无边界性

    • 其安全威胁超越了现实地域和现实行业的限制

  • 非传统性

    • 与军事安全、政治安全等传统安全相比，其涉及领域和影响范围广泛

信息系统安全

  • 信息系统安全
    • 设备安全
      • 其安全是信息系统安全的首要问题
      • 包括：设备的稳定性、可靠性、可用性
      • 设备：硬件设备和软件设备
    • 数据安全
      • IBM对其定义：采取措施确保数据免受未授权的泄露、篡改和毁坏
      • 包括：数据的秘密性、真实性、完整性
    • 内容安全
      • 是信息安全在法律、政治、道德层次上的要求
      • 包括：政治上健康、符合国家法律法规、符合道德规范
    • 行为安全
      • 是信息安全的终极目的
      • 包括：行为的秘密性、完整性、可控性
      • 即哲学上，实践是检验真理的唯一标准

1.2-信息安全威胁

我国面临的信息安全威胁

  • 从受威胁的对象，分3类
    • 国家威胁
    • 组织威胁
    • 个人威胁
  • 国家威胁
    1. 恐怖组织，发布恐怖信息，渲染暴力活动
    2. 邪教组织，宣扬种族歧视，宣传邪教理念等
    3. 西方势力，传播西方意识形态、价值观和生活方式，进行文化渗透、侵略
    4. 其他国家情报机构，收集我国政治、军事等情报信息
  • 组织威胁
    • 针对企业或组织受保护的财产、专有技术
    • 具体表现为：
      • 网络恐怖分子破坏公共秩序，制造社会混乱
      • 工业间谍掠夺竞争优势，打击竞争对手
  • 个人威胁
    • 一是对知识产权的威胁
    • 二是侵犯、破坏个人计算机系统中信息，对其财产权进行侵犯；或对email进行破坏，影响正常生活学习工作

信息安全问题产生的根源

  • 一般认为
    • 病毒、漏洞、黑客的存在
    • 都是原因，而非根源
  • 内因
    • 系统自身脆弱性
  • 外因
    • 人为威胁
    • 环境威胁

内因相关

  • 内在结构的复杂
    • 包括：工作站中存在信息数据、员工、移动介质、网络中其他系统、网络中其他资源、访问Internet、访问其他局域网、到Internet的其他路由、电话和调制解调器、开放的网络端口、远程用户、厂商和合同方的访问访问外部资源、公共信息服务、运行维护环境
  • 内因示例
    • 复杂性导致脆弱性
    • 凡是人做的东西总会存在问题
    • 网络太复杂、软件太复杂、应用太复杂

外因相关

  • 国家安全威胁
    • 信息战士：制造混乱，进行目标破坏
    • 情报机构：搜集政治、军事等信息
  • 组织威胁
    • 恐怖分子：破坏公共秩序
    • 商业间谍：掠夺竞争优势
    • 犯罪团伙：施行报复，实现经济目的
  • 局部威胁
    • 社会性黑客：攫取金钱，恐吓
    • 娱乐性黑客：吓人为乐
  • 来自大自然的威胁
    • 地震等自然灾害
    • 电力、空调等被破坏
    • 信息系统机房和设备受破坏

1.3-信息安全发展阶段与形势

信息安全的发展阶段

  • 可分（按信息技术的发展阶段来分）：

    1. 通信安全（COMSEC）
    2. 计算机安全（COMPUSEC）
    3. 信息系统安全（INFOSEC）
    4. 信息安全保障（IA）

  • 通信安全

    • communication security，20世纪，40年代-70年代
    • 核心思想：
      • 通过密码技术解决通信保密，保证数据的保密性和完整性
      • 主要关注传输过程中的数据保护
    • 安全威胁
      • 搭线窃听
      • 密码学分析
    • 安全措施
      • 加密

  • 计算机安全

    • computer security，20世纪，70-90年代
    • 核心思想
      • 确保信息系统的保密性、完整性和可用性
    • 安全威胁
      • 非法访问
      • 恶意代码
      • 脆弱口令
    • 安全措施
      • 安全操作系统设计技术（TCB）

  • 信息系统安全

    • information system security，20世纪，90后
    • 核心思想
      • 确保信息在存储、处理和传输过程中免受偶然或恶意的泄密、非法访问或破坏
    • 安全威胁
      • 网络入侵
      • 病毒破坏
      • 信息对抗
    • 安全措施
      • 防火墙
      • 防病毒
      • 漏洞扫描
      • 入侵检测
      • PKI
      • VPN

  • 信息安全保障

    • information assurance，今天，将来

    • 核心思想

      • 动态安全，保障信息系统的业务正常、稳定的运行
      • 综合技术、管理、过程、人员

    • 安全威胁

      • 黑客
      • 恐怖分子
      • 信息战
      • 自然灾害
      • 电力中断

    • 安全措施

      • 技术安全保障体系
      • 安全管理体系
      • 人员意识/培训/教育

我国信息安全形势

  • 13年，棱镜门事件，反映出互联网背后的国家力量；网络安全成为大国竞争的战略基点
  • 我国：
    • 针对网络信息的破坏日益严重，利用网络进行违法犯罪的案件逐年上升
    • 安全漏洞和安全隐患增多，对信息安全构成威胁
    • 黑客攻击、恶意代码对信息安全系统造成影响

1.4-信息安全保障

含义

  • 简介

    • 采用技术、管理等综合手段，保护信息和信息系统能够安全运行的防护性行为
    • 通过保证信息和信息系统的可用性、完整性、机密性和不可否认性来保护并防御
    • 包括通过综合保护、检测和响应等能力为信息系统提供修复

  • 具体措施：

    • 防止信息泄露修改和破坏
    • 检测入侵行为，计划和部署针对入侵行为的防御措施
    • 采用安全措施和容错机制在遭受攻击的情况下保证机密性、私密性、完整性、抗抵赖性、真实性、可用性和可靠性
    • 修复信息和信息系统所遭受的破坏

  • 信息安全保障是一种立体保障

  • 与信息安全、信息系统安全的区别

    （信息安全保障的概念更加广泛）

    • 信息安全的重点是保护和防御，而安全保障的的重点时保护、检验和响应综合
    • 信息安全不太关注检测和响应，但是信息安全保障却关注
    • 攻击后的修复不在信息安全概念的范围之内，但是它是信息安全保障的重要组成部分
    • 信息安全的目的是为了防止攻击的发生，而信息安全保障的目的是为了信息系统始终能保持特定水平的可用性、完整性、真实性机密性和抗抵赖性

模型

  • 简介
    • 计划-执行-检查-改进（plan do check act，PDCA）模型和信息保障技术框架是基本方法和思想
    • 防护-检测-响应（protection detection response，PDR）模型
    • 策略-防护-检测-响应（policy protection detection response，P2DR/PPDR）模型
  • PDR
    • 基本思想：
      • 承认漏洞存在，正视系统的威胁
      • 通过适度防护，并加强检测
      • 落实安全事件响应，建立威胁源威慑，保证安全
    • 其认为：
      • 任何防护措施都是基于时间的，超过该事件段，措施即可能被攻破
    • 优缺点：
      • 优：直观、实用
      • 缺：对隐患及措施采取相对固定的假设前提，难以适用网络环境的变化
  • P2DR/PPDR
    • 所有防护、检测、响应都是依据策略实施的
    • 动态性

作用

  • 构建安全可靠、有用、可信的互联网，服务于和谐社会
  • 国家战略，信息化与信息安全协调发展
  • 确保信息化带来经济与社会效益，有助于信息化发展
  • 是保障和促进信息化发展，而非牺牲信息化发展来换取信安
  • 采取不上网、不共享、不互联互通等传统封闭的方式保障安全是不可取的
  • 信息化发展是解决信息安全问题的技术和物质保障
  • 为推进信息化发展，要高度重视安全建设
  • 我国政府提出互联网管理基本原则，要依法

1.5-信息系统安全保障

信息系统是什么

  • 信息系统是具有集成性的系统，每个组织中信息的流动总和构成一个信息系统

  • 是根据一定需要进行输入、系统控制、数据处理等活动而涉及到的所有因素的综合体

信息系统安全保障的含义

  • 含义：

    • 在信息系统的整个生命周期
    • 从技术管理工程人员等方面提出安全保障要求
    • 确保信息系统的保密性、完整性和可用性
    • 降低风险到可接受的程度，从而保障系统实现组织机构的使命

  • 信息系统安全保障相关概念和关系

    • 风险
      • 产生因素：系统自身存在的漏洞和系统外部的威胁
      • 信息系统运行环境中存在具有特定威胁动机的威胁源，通过各种攻击方法，利用其脆弱性，造成不良影响，引发安全问题
    • 保障
      • 为了降低风险
      • 针对风险，制定策略，采取措施，降低风险，实现其业务使命
      • 也要遵循经济原则
    • 使命
      • 描述了整个生命周期运行的需求和目标
      • 使命与其安全保障密不可分，通过安全措施保障目标正确执行
      • 随着威胁及运行环境的变化，安全保障需提供保障措施，以确保系统正常
