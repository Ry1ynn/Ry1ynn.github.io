2.1-密码学

威胁、安全属性与密码学技术

  • 如下表：

    | 攻击威胁 | 所破坏的属性 | 解决问题所采用的密码学技术 |
    | —————————- | ———— | ———————————— |
    | 截获（泄露信息） | 机密性 | 对称密码和非对称密码 |
    | 篡改（修改信息） | 完整性 | 哈希函数、数字签名、对称和非对称密码 |
    | 伪造（伪造信息来源 | 真实性 | 数字签名 |
    | 否认（事后否认发送信息和行为 | 不可否认行 | 数字签名 |

密码学发展

  1. 第一：古代-19世纪末，古典密码
  2. 第二：20世纪初-1949，近代密码
  3. 第三：从香农于1949年发表的划时代论文‘the communication theory of secret system’开始，现代密码
  4. 第四：从1976W.Diffie和M.Hellman发表论文’New Directions in Cryptography’开始，公钥密码
  • 阶段：
    • 古典密码
    • 近代密码
    • 现代密码
    • 公钥密码

古典密码学

  • 古典密码体制的安全性在于：
    • 保持算法本身的保密性
  • 受到算法限制：
    • 不适合大规模生产
    • 不适合较大的或者人员变动较大的组织
    • 用户无法了解算法的安全性

近代密码学

  • 20世纪初-1949
  • 主要标志：
    • 机械密码/机电密码
    • 用机电代替手工
  • 近代密码体制是用机械或电动机械实现的，最著名的如转轮机（Rotor machine）
    • 转轮密码及enigma，二战期间，德国最高级密码机
    • 英国typex打字密码机

现在密码学

  • 1949-1975
  • 新特点：
    • 数据的安全是基于密钥而不是算法的保密

公钥密码学

  • 1976年以后
  • 使得发送端和接收端无密钥传输的保密通信成为可能

加密与解密

  • 明文（plaintext ）：原始消息，被屏蔽消息，未经加密的消息

  • 密文（Ciphertext）或密报（cryptogram）：明文经密码变换而成的一种隐蔽形式

  • 加密员或密码员（cryptography）：对密码进行加密操作的人员

  • 截获：

    • 一个非授权方介入系统，窃听传输的信息，导致信息泄露，

    • 破坏密码保密性

    • 非授权方可是人，可是程序

    • 截获攻击主要包括：

      • 通过嗅探和监听等手段截获信息，从而推测有用信息
      • 如用户口令、账号、文件或程序的不正当复制

  • 加密，encryption

    • 明文变为密文的过程
    • 可懂的语言变换成人类或机器不可懂得语言

  • 解密，Decryption

    • 密文恢复出原明文的过程
    • 加密的逆过程
    • 把不可懂语言变换为可懂的语言

对称密码算法

加密可采用密码算法实现，密码算法从密钥使用角度，可分为对称密码算法和非对称密码算法

  • 也称单钥或私钥密码算法

  • 加密密钥和解密密钥相同（或实质上相同，即从一个易推出另一个）

  • 如，DES，3DES，IDEA，AES

  • 密钥相对较短，一般采用128，192或256比特

  • 优点：

    • 效率高，算法简单，系统开销小
    • 适合加密大量数据
    • 明文长度与密文长度相等

  • 缺点：

    • 需要以安全方式进行密钥交换
    • 密钥管理复杂
    • 无法解决消息的篡改、否认等问题

非对称密码算法

  • 又叫公钥密码算法，public-key cipher
  • 加密密钥和解密密钥不同（或者从一个很难推出另一个）
    • 对外公开的密钥，公开密钥，公钥，public key
    • 必须保密的，私有密钥，私钥，private key
  • 如，RSA，ECC，ElGamal
  • 公钥加密，私钥解密（或私钥加密，公钥解密）

  • 密钥长度512-2048位

非对称密码学特点

  • 优势：
    • 密钥分发数目与参与者数目相同
    • 在有大量参与者的情况下易于密钥管理
    • 安全
    • 支持数字签名和不可否认性
    • 无需事先与对方建立关系，交换密钥
  • 缺点：
    • 速度相对较慢
    • 可能比同等强度的对称密码算法慢10到100倍
    • 加密后，密文变长

哈希函数

  • 对称密码和非对称密码算法主要解决信息的机密性问题，而实际系统和网络还可能受到消息篡改等记录
    • 篡改攻击包括：修改消息内容，改变程序使其不能正确运行等
  • 哈希函数可用来保证信息的完整性
  • 哈希函数（HASH函数），也称为杂凑函数或单向散列函数
  • 接受一个消息作为输入，产生一个称为哈希值的输出，输出的哈希值也可称为散列值，消息摘要（message digest，MD）

MD5算法

  • md，message digest，消息摘要
  • 输入：任意长度的消息
  • 输出：128位消息摘要
  • 处理：以512位输入数据块为单位

SHA-1 算法

  • sha，secure hash algorithm，安全哈希算法
  • 输入：最大长度为264位的 消息
  • 输出：160位消息摘要
  • 处理：输入以512位数据块为单位处理

比较sha1和md5

  • 散列值长度
    • MD5 128位
    • SHA1，160位
  • 安全性
    • sha1算法被视为MD5的替代候选法
  • 速度
    • sha1慢些

数字签名

  • 除了保护信息的机密性和完整性，密码学技术也可以解决信息的可鉴别性（真实性或认证性）和不可否认性

  • 数字签名的基本特点

    • 能与被签的文件在物理上不可分割
    • 签名者不可否认自己的签名
    • 签名不能被伪造
    • 容易被验证

  • 数字签名是传统物理签名的数字化，基本要求

    • 能与所签文件绑定
    • 签名者不能否认自己的签名
    • 签名不能被伪造
    • 容易被验证

数字签名的应用

  • 可信性：

    • 签名让文件接收者相信签名者是慎重的在文件上签名的

  • 不可重用性：

    • 签名不可重用，即使同意消息在不同时刻的签名也是有有区别的；
    • 如果将签名部分提取出来，附加在别的消息后面，验证签名会失败
    • 因为签名和所签名消息是一一对应的，消息不同签名内容也不同，因此签名无法重复使用

  • 数据完整性

    • 文件签名后，文件不能改变

  • 不可伪造性

    • 签名能证明签名者而不是其他人在文件上签名，任何人都不能伪造签名

  • 不可否认性

    • 在签名者否认自己签名时，签名接受这可以请求可信第二方进行仲裁

2.2-数字证书和公钥基础设施

数字证书

  • 绑定用户身份和公钥
  • 网络世界的电子身份证
    • 与现实世界的类似
    • 能证明个人、团体或设备的身份
  • 包含相关信息
    • 姓名、地址、等信息
    • 所有者的公钥
  • 拥有者拥有证书公钥对应的私钥
  • 由可信的颁发机构颁发
    • 如身份证由公安局颁发
  • 颁发机构对证书进行签名
    • 与身份证上的公安局的盖章类似
    • 可以由颁发机构证明证书是否有效
    • 可防止擅改证书上的任何信息

公钥基础设施

  • PKI，public key infrastructure
  • 利用公开密钥技术建立的提供信息安全服务的在线基础设施
  • 利用加密、数字签名、数字证书来保护应用、通信等安全
  • 一个包括硬件、软件、人员、策略和规程的集合
  • 实现基于公钥密码体制的密钥和数字证书的产生、管理、存储等
  • 如同电力基础设施为家用电器提供电力一样，pki为各种应用提供安全保障，提供网络信任基础

CA:认证权威机构

  • 简介：

    • CA专门负责数字证书的产生、发放和管理，以保证数字证书的真实可靠
    • 是 PKI的核心
    • 负责管理所有用户证书，把用户的公钥和用户的其他信息捆绑在一起，在网上验证用户的身份

  • 主要功能

    • 签发数字证书
      • 签发证书
      • 更新证书
    • 管理数字证书
      • 撤销、查询
      • 审计、统计
    • 验证数字证书
      • 黑名单验证（CRL
    • RA的设立、审查及管理

RA：证书注册机构

  • 证书注册权威，Registration Authority
  • 受理用户的数字证书申请
    • 对证书申请者身份进行审核并提交CA制证
    • 类似与申请身份证的派出所
  • 提供证书生命期的维护工作
    • 受理用户证书申请
    • 协助颁发用户证书
    • 审核用户真实身份
    • 受理证书更新请求
    • 受理证书吊销

目录服务（LDAP）

  • 证书的存储库，提供了证书的保存、修改、删除和获取的能力
  • CA采用LDAP标准的目录服务存放证书，其作用与数据库相同，优点是在修改操作少的情况下，对于访问的效率比传统数据库要高

CRL

  • CRL，certificates revocation list，证书撤销列表，也成证书黑名单
  • 在证书的有效期期间，因为某种原因（如人员调动，私钥泄露等），导致相应的的数字证书内容不再是真实可信，此时，进行证书撤销，说明该证书已经失效
  • CRL中列出了被撤销的证书序列号

证书载体

  • 内存、IC卡、软盘……
  • USB-KEY

2.3-身份认证

身份认证基本方法

  • 身份认证是用户登录系统或网站的第一道安全防线，如输入账号口令来登录，
  • 身份认证是在网络中确认操作者身份的过程
  • 身份认证一般依据以下三种基本情况或三种情况的组合来鉴别用户身份
    • 用户所知道的东西：如口令，密钥
    • 用户拥有的东西，如印章，U盾（USB key）
    • 用户所具有的生物特征，如指纹，声音，人脸等

静态口令认证

  • 根据“用户所知”进行认证的方法有：静态口令认证、短信口令认证、和动态口令认证等

  • 静态口令

    • 口令固定不变，并且长期有效

    • 弱鉴别（weak authentication）

    静态口令认证缺点

  • 通信窃取

    • 窃听获得明文的用户名和口令
    • 重放攻击-截获登录数据（明/密），重放攻击
  • 字典攻击
    • 选择有意义的单词或数字作字典
  • 暴力攻击
    • 穷举全部可能组合猜测口令
  • 外部泄露
    • 搜索存有口令的纸片或文件
  • 窥探
    • 安装监视器或从背后窥探
  • 社交工程
    • 冒充合法用户、假冒管理员，骗取口令

短信口令认证

  • 利用移动网络获取动态口令的认证方式
  • 以手机短信方式请求包含6位随机数的动态口令，用户认证跟系统以短信形式发送随机的的6位动态口令到用户的手机上，用户在认证是输出口令即可
  • 由于手机与用户绑定较紧密，短信口令生成与使用场合是物理哥觉得，因此口令在通路上被截取的几率较低

动态口令认证

  • 动态口令，又称一次性口令，one time password，OTP
  • 口令动态性：每次变化，无需人工干预
  • 口令随机性：随机性强，难以猜测

动态口令应用示例

  • 刮刮卡
  • 动态令牌

U盾，usb-key

  • 根据用户所有进行认证的方法有U盾、电子印章
  • usb-key认证，近几年发展起来的一种方便、安全的身份认证技术
  • 采用软硬件结合的挑战/应答认证模式
  • usb-key是一种usb接口的硬件设备，其内置单片机或智能卡芯片，可存储用户的密钥或数字证书，利用usb-key内置的密码算法实现对用户身份的认证
  • 挑战/应答认证模式，即认证系统发送一个随机书（挑战），用户使用usb-key中的密钥和算法计算出一个数值，认证系统对该数值进行检验，若正确则认为为合法用户

基于生物特征的身份认证

  • 每个人所具有的唯一生理特征
    • 指纹、视网膜、声音等
  • 指纹
    • 一些曲线和分叉及一些非常微小的特征
    • 提取指纹中的一些特征并且存储这些特征信息，节省资源，快速查询
  • 手掌、手型
    • 手掌有折痕，起皱，还有凹槽
    • 包括每个手指的指纹
    • 人手的形状，表示了手的几何特征
  • 视网膜扫描
    • 扫描眼球后方的视网膜上面的血管的图案
  • 虹膜扫描
    • 虹膜是眼睛中位于瞳孔周围的一圈彩色的部分
    • 虹膜有其独有的方案，分叉，颜色，等
  • 语音识别
    • 记录时说几个不同的单词，然后识别系统将这些单词混杂杂一起，让其再次读出给出的一系列单词
  • 面部扫描
    • 人都有不同的骨骼结构

指纹识别的实现原理

  • 通过特殊的光电扫描和计算机图像处理技术，对指纹进行采集、分析和对比等，自动、迅速、准确的认证出个人身份

  • 指纹识别的过程：

    • 按照用户的和姓名等信息将其存在指纹数据库中的模板指纹调出来，然后再用用户输入的指纹与该模板的指纹相匹配，以确定这两幅指纹是否出自同一幅指纹

虹膜识别的实现原理

  • 虹膜时环绕在瞳孔四周有色彩的部分
  • 每个虹膜有独特结构
  • 每个人虹膜各不相同，左右眼也可能不一样
  • 虹膜成型后不再发生改变（出生后6-18月）

其他身份认证技术

  • 单点登录
    • SSO,single-sign on
    • 用户只需要在登录时进行一次注册，就可以访问多个系统，不必重复输入用户名和密码来确定身份
  • Kerberos
    • 提供了一个网络环境下的身份认证框架结构
    • 实现采用对称密钥加密技术
    • 公开发布的版本有版本4和版本5
    • 安全性、可靠性、可伸缩性、透明性
  • 远程用户拨号认证系统（RADIUS）
    • 是一种C/S结构的协议，客户端最初时NAS服务器，现在任意运行RADIUS客户端软件的计算机都可以成为RADIUS的客户端
    • 基本设计组件有认证、授权和记账（AAA）
    • 简单精确，可扩充
  • 使用UDP端口，1812，1813

2.4-访问控制

访问控制的概念和目标

  • 访问控制：针对越权使用资源的防御措施
  • 目标：
    • 防止对任何资源（如计算资源、通信资源或信息资源）进行未授权的访问
    • 从而使资源在授权范围内使用，决定用户能做什么，也决定代表一定用户利益的程序能做什么

访问控制的作用

  • 未授权访问：
    • 包括未经授权的使用、泄露、修改等
    • 非法用户对系统资源的使用
    • 合法用户对系统资源的非法使用
  • 作用
    • 机密性，完整性和可用性

主体与客体

  • 主体
    • 发起者，是一个主动的实体，可以操作被动实体的相关信息或数据
    • 用户、进程、程序等
  • 客体
    • 一种被动实体，被操作的对象，规定需要保护的资源
    • 文件、存储介质、程序、进程等
  • 主体：接受客体相关数据和信息，也可能改变客体的相关信息
  • 一个主体为了完成任务，可以创建另外的主体，这些自主体可以在网络上不同的计算机上运行，并由父主体控制他们
  • 客体：始终是提供、驻留信息或数据的实体
  • 主体和客体的关系是相对的，角色额可以互换

控制策略

  • 是主体对客体的相关访问规则集合，即属性集合，访问策略体现了一种授权行为，也是客体对主体某些操作行为的默认
  • 访问控制是主体依据某些控制策略或访问权限，对客体本身或其资源赋予不同访问权限的能力，从而保障数据资源在合法范围内得以有效使用和管理
  • 访问控制安全策略实施遵循最小特权原则，在主体执行操作是，按照主体所需权力的最小化原则分配给主体权利。最大限度地限制主体实施授权行为，避免突发事件、操作错误和未授权主体等意外情况可能给系统造成的危险

授权

  • 规定主体可以对客体执行的操作
    • 读、写、执行、拒绝访问

自主访问控制

  • DAC，是一种广泛应用的访问控制方法
  • 采用这种方法，资源的所有者（往往也是创建者）可以规定谁有权访问他们的资源，用户（或用户进程）可以选择与其他用户共享资源

访问控制矩阵

  • 矩阵元素：规定了相应用户对应于相应的文件被准予的访问许可、访问权限

  • 图示：

自主访问控制的特点

  • 优点：
    • 根据主体的身份和访问权限进行决策
    • 具有某种访问能力的主体能够自主地将访问权的某个子集授予其他主机
    • 灵活性高，被大量采用
  • 缺点：
    • 信息在传递过程中其访问权限关系会被改变

强制访问控制的含义

  • 强制访问控制MAC，是指主体和客体都有一个固定的安全属性，系统用该安全属性来决定一个主体是否可以访问某个客体，如果系统认为具有某一个安全属性的主体不适合访问某个客体，那么任何其他主体都无法使该主体具有访问该客体的权利
  • 主体和客体分配有一个安全属性
  • 应用于军事等安全要求较高的的系统
  • 可与自主访问控制结合使用

自主访问控制与强制访问控制的比较

  • 自主访问控制
    • 细粒高
    • 灵活性高
    • 配置效率低
  • 强制访问控制
    • 控制粒度大
    • 灵活性不高
    • 安全性强

基于角色的访问控制（RBAC

  • RBAC的基本思想是根据用户所担任的角色来决定用户在系统中的访问权限

  • 一个用户必须扮演某种角色，而且还必须激活这一角色，才能对一个对象进行访问或执行某种操作

  • 图示

RBAC模型的特点

  • 便于授权管理（角色的变动远远低于个体的变动）
  • 便于处理工作分级，如文件等资源分级管理
  • 利用安全约束，容易实现各种安全策略，如最小特权、职责分离等
  • 便于任务分担，不同角色完成不同任务

2.5-安全审计

  • 即便是采用了很多方法来保障计算机或网络安全，但仍然不可避免系统感染病毒或数据安全受到威胁。受到威胁后，如何从安全事故中恢复过来，使系统尽快运作起来，需要预先采取一些措施
  • 安全审计是提高安全性的重要工具，它能够再现问题，以帮助时候的责任追查和数据恢复等
  • 审计技术的的出现早于计算机技术，它按照时间顺序产生、记录系统事件，并对其进行检查。
  • 安全审计可以跟踪和监测系统中的异常事件，也可以监视系统中其他安全机制的运行情况
  • 计算机审计，Audit是按照一定的安全策略，记录历史操作事件，并利用记录进行分析，发现系统漏洞，入侵行为等，并改善系统性能和安全性的一系列过程
  • 安全审计是对访问控制的必要补充用，他会对用于使用何种信息资源、使用的事件，以及如何使用（执行何种操作）进行记录和监控
  • 通过对系统和用户进行充分和适当审计，能够分析发现安全事件的原因，并提供相应的证据
  • 日志是安全审计系统的主要组成部分。为了维护自身系统资源的运行状况，计算机系统一般会有相应的日志系统，记录有关日常事件或误操作警报的日期及时间
  • 受到可疑攻击或安全威胁后，可以通过查看事件安全日志来确认可疑或恶意的行为，
  • 例如，通过查看事件日志，发现非工作事件某个用户成功登录账户，则可能有人切确了账号和口令
  • 日志中有多次登录失败的记录，则有可能有攻击者尝试进入系统

  • 图示

  • 安全审计可以分为被动式审计和主动式审计两种
  • 被动审计是简单记录一些活动，并不做处理
  • 主动式审计一般包括：结束一个登录会话、拒绝一些主机的访问（包括web站点，FTP服务器和电子邮件服务器）、跟踪非法活动的源位置等行为
  • 安全审计的四个作用：
    • 威慑和警告潜在的攻击者和滥用授权的合法用户。
    • 提供有价值的系统使用日志，帮助系统管理员及时发现系统入侵行为额或潜在的系统漏洞
    • 发生故障后，可以帮助评估故障损失、重建事件和数据恢复
    • 对系统控制、安全策略与规程中特定的改变做出评价和反馈，便于修订决策和部署
